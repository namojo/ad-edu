# Media Harness — 미디어·캠페인 플래닝 하네스 (감독자)

중앙의 **media-planner(감독자)** 가 캠페인을 타겟·채널믹스·예산배분·A/B로 나눠 지휘하고, 필요한 근거는 researcher에게, 최종 산출은 doc-producer(Excel)에게 맡겨 취합하는 오케스트레이터(팀장)입니다. 감독자가 상태를 관리하며 동적으로 태스크를 분배하는 것이 이 패턴의 핵심입니다.

**실행 모드:** 에이전트 팀(기본). 감독자가 중앙에서 지휘하고, 하위 작업을 상황에 따라 배분·취합합니다. 모든 에이전트는 `model: opus`로 호출합니다.

## 팀 구성 (에이전트 풀에서)

| 팀원(에이전트) | 역할 | 사용 매뉴얼(스킬) | 출력 |
|---|---|---|---|
| media-planner | **감독자** — 타겟/채널/예산/A/B 분해·배분·취합 | media-planning | `_workspace/2_mediaplanner_plan.md` |
| researcher | 타겟·채널 벤치마크 등 근거 조사(감독자가 요청 시) | market-research | `_workspace/1_researcher_bench.md` |
| doc-producer | 채널×예산×KPI 미디어 플랜 Excel 산출 | doc-automation (xlsx) | `_workspace/3_docproducer_mediaplan.xlsx` |

## Phase 0: 컨텍스트 확인 (초기/후속/부분 재실행 판별)

1. `_workspace/` 존재 여부 확인.
2. 실행 모드 결정:
   - **없음** → 초기 실행. Phase 1로.
   - **있음 + 부분 수정 요청**(예: "예산만 재배분", "공격형 시나리오로", "엑셀만 다시") → 부분 재실행. 감독자가 해당 하위 태스크만 다시 배분하고, 바뀐 부분의 파일만 덮어쓴다.
   - **있음 + 새 캠페인/새 예산** → 새 실행. 기존 `_workspace/`를 타임스탬프 폴더로 이동 후 새로 시작.
3. 부분 재실행 시 이전 플랜(`2_mediaplanner_plan.md`)을 감독자에게 읽혀 델타만 반영시킨다.

## Phase 흐름 (감독자 지휘)

### Phase 1: 준비
1. 브리프에서 **총 예산, 목표(KPI), 기간, 타겟**을 뽑아 `_workspace/00_input/brief.md`로 저장한다.
2. 예산 합계·목표를 감독자에게 전달할 형태로 정리한다.

### Phase 2: 팀 구성
1. `media-team`을 만들고 media-planner(감독자), researcher, doc-producer를 `model: opus`로 등록.
2. 감독자에게는 "큰 목표만 주고 세부는 위임" 원칙을 지시한다(과도한 세부 지시 금지).

### Phase 3: 감독자 지휘 — 태스크 분해·배분
**실행 방식:** media-planner가 중앙에서 상태를 관리하며 하위 태스크를 동적으로 배분한다.
1. 감독자가 캠페인을 **타겟 정의 → 채널믹스 → 예산배분 → A/B 계획** 4개 태스크로 분해한다.
2. 근거가 필요한 태스크(예: 채널별 CPM 벤치마크, 타겟 규모)는 researcher에게 SendMessage로 조사 요청 → researcher가 `_workspace/1_researcher_bench.md`로 회신.
3. 감독자가 조사 근거를 받아 채널믹스·예산배분을 확정하고, **합계 검산**(채널 예산 합 = 총 예산)을 반드시 수행한다.
4. A/B 계획(가설·변인·성공지표)을 포함해 통합 플랜을 `_workspace/2_mediaplanner_plan.md`로 저장한다.

### Phase 4: 산출물화 — Excel
1. doc-producer가 감독자 플랜을 읽어 **채널 × 예산 × 예상 KPI** 미디어 플랜 시트를 xlsx로 만든다.
2. 합계 행에서 예산 총합이 브리프와 일치하는지 검산 서식을 넣는다.
3. `_workspace/3_docproducer_mediaplan.xlsx`로 저장.

### Phase 5: 마무리
1. 최종 산출물을 지정 경로로 내보낸다: 미디어 플랜 `course4-media/사례/C4_제로톡_미디어플랜.xlsx`, 캠페인 브리프 `course4-media/사례/C4_제로톡_캠페인브리프.md`.
2. 요청 시 보수형/공격형 시나리오 비교를 감독자에게 재지시(부분 재실행)함을 안내한다.
3. 팀 정리, `_workspace/` 보존.

## 데이터 전달

- **팀 모드:** 태스크 + 파일 + 메시지. 감독자가 조사 요청·결과 취합을 SendMessage로 조율하고, 각 하위 산출물은 파일로 남긴다.
- **파일 규약:** `_workspace/{phase}_{agent}_{artifact}.{ext}`.
- **최종 산출물만** `course4-media/사례/`로 내보낸다(Excel + 브리프).

## 데이터 흐름(그림)

```
                          [media-planner 감독자]
                          │   분해·배분·취합
        ┌────────────────┼──────────────────┐
        ▼(근거 요청)       ▼(직접 확정)          ▼(산출 위임)
  [researcher]        타겟/채널믹스/         [doc-producer]
  1_researcher_bench   예산배분/A/B          3_docproducer_mediaplan.xlsx
        └───SendMessage─▶ 2_mediaplanner_plan.md ─▶ Read
                                                     ▼
                          course4-media/사례/ (Excel + 캠페인 브리프)
```

> **비결정성 안내:** 감독자는 매번 태스크를 조금 다르게 배분할 수 있습니다. 중요한 것은 큰 목표(예산·KPI)만 주고 세부는 위임하는 것입니다.
> **검증 습관:** 채널 예산의 합이 총 예산과 일치하는지 반드시 검산하세요. 합계가 어긋난 미디어 플랜은 신뢰를 잃습니다.

## 에러 핸들링

| 상황 | 대응 |
|---|---|
| 예산 합계 불일치 | 감독자가 배분표 합계를 재검산, 초과/미달분을 조정해 재저장 |
| KPI 없는 채널 배분 | 감독자가 채널별 목표지표를 반드시 채우도록 재지시 |
| 조사 근거 미확보 | researcher 1회 재시도 → 실패 시 "벤치마크 미확보" 명시, 감독자가 보수적 가정으로 진행 |
| 감독자에게 과도한 세부 지시 | 큰 목표만 주고 위임 원칙으로 복귀 |
| Excel 합계 오류 | doc-producer가 검산 서식으로 재생성 |

## 팀 크기

**중규모(3명 코어).** 감독자 1 + 조사 1 + 문서 1이 기본. 채널 카테고리가 많으면 감독자가 채널별 하위 태스크를 늘려 배분하되, 코어 팀원 수는 유지한다.

## 테스트 시나리오

### 정상 흐름
1. 사용자가 "제로톡" 런칭 캠페인, 총 예산 5,000만원, 목표 KPI를 입력.
2. 감독자가 타겟/채널/예산/A/B로 분해, 필요한 벤치마크를 researcher에 요청.
3. 감독자가 채널믹스·예산배분 확정(합계 검산 통과), A/B 계획 포함해 플랜 저장.
4. doc-producer가 채널×예산×KPI Excel 생성.
5. 최종 `course4-media/사례/C4_제로톡_미디어플랜.xlsx` + 브리프 내보냄.

### 에러 흐름
1. Phase 3에서 채널 예산 합이 총 예산을 초과.
2. 감독자가 합계 검산에서 초과 감지 → 배분 재조정.
3. researcher의 특정 채널 CPM 벤치마크 미확보 시 "보수적 가정 적용" 명시.
4. 최종 보고에 가정·미확보 항목 표기.

## 팀장 완주 체크

- [ ] 감독자에게 큰 목표(예산·KPI·기간)만 주고 세부는 위임했는가
- [ ] 타겟/채널믹스/예산배분/A/B 4개로 분해·취합됐는가
- [ ] 채널 예산 합계가 총 예산과 일치하는지 검산했는가
- [ ] 채널마다 목표 지표(KPI)가 채워졌는가
- [ ] Excel 미디어 플랜과 브리프만 `course4-media/사례/`로 내보냈는가

## 실습 트리거 대표 프롬프트

```text
미디어·캠페인 플래닝 하네스를 구성해줘. "제로톡" 런칭 캠페인(총 예산 5,000만원)을 중앙에서 지휘하는 감독자 에이전트가 타겟 정의, 채널 믹스, 예산 배분, A/B 테스트 계획을 각 전문가에게 나눠 맡기고 취합해줘. '채널별 예산 배분표(엑셀), 타겟 정의, A/B 테스트 계획'이 담긴 미디어 플랜을 만들어줘.
```
