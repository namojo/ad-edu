# 인사이트 종합 (Insight Synthesis)

조사는 사실의 더미다. 인사이트는 그 더미에서 **"그래서 우리가 뭘 해야 하는가"** 를 끌어내는 일이다. 여러 조사를 교차검증해 종합한다.

## 왜 이 매뉴얼인가

조사 결과를 그냥 이어 붙이면 요약일 뿐 인사이트가 아니다. 인사이트가 실패하는 이유는 셋이다. (1) 한 자료만 보고 성급히 결론낸다. (2) 주장만 있고 근거가 없어 설득이 안 된다. (3) "소비자는 건강을 중시한다" 같은 뻔한 말에서 멈춰 실행으로 안 이어진다. 이 매뉴얼은 **교차검증 → 주장→근거→시사점** 이라는 뼈대로 이 셋을 막는다. 좋은 인사이트는 "당연한 말"이 아니라 "그래서 이렇게 움직이자"가 나오는 말이다.

## 워크플로우

### 1단계 — 입력 조사 모으고 정렬

종합할 조사들을 한자리에 놓는다. 시장·소비자·경쟁·트렌드 등 서로 다른 축의 자료일수록 좋다. 각 자료의 **범위(카테고리·기간·지역)와 출처**를 먼저 확인한다. 범위가 다른 자료를 같은 저울에 올리면 왜곡된다. (예: 해외 트렌드와 국내 소비자 데이터를 섞으려면 그 차이를 감안한다.)

### 2단계 — 교차검증 (신호 vs 잡음 가르기)

각 발견을 다른 자료와 대조한다. 이 단계가 인사이트의 신뢰를 만든다.

- **여러 자료가 같은 방향** → 신호가 강하다. 확신 있게 인사이트로 승격.
- **한 자료에만 있음** → 약한 신호. "단일 출처"라고 표기하고 조심스럽게 다룬다.
- **자료끼리 충돌** → 숨기지 말고 드러낸다. 왜 다른지(집계 기준·시점·지역 차이) 따져보고, 그래도 안 풀리면 양쪽을 함께 남긴다.

> 교차검증은 다수결이 아니다. 3개 자료가 같은 말을 해도 출처가 한 곳에서 파생됐으면 신호는 하나다. 출처의 독립성을 확인한다.

### 3단계 — 인사이트 도출: 주장→근거→시사점

이 삼단 구조가 이 매뉴얼의 심장이다. 인사이트 하나마다 이 셋을 모두 채운다.

- **주장 (Claim)**: 한 문장의 단언. 뻔하지 않고 방향성이 있어야 한다.
- **근거 (Evidence)**: 그 주장을 뒷받침하는 조사 사실 + 출처. 교차검증을 통과한 것.
- **시사점 (So-what)**: "그래서 캠페인/제품/메시지를 이렇게" 라는 실행 함의.

**나쁜 예 (주장뿐):** "소비자는 건강에 관심이 많다."

**좋은 예 (삼단):**
```
주장: 이 카테고리 구매자는 '건강'보다 '죄책감 없는 즐거움'에 반응한다.
근거: 소비자 조사에서 구매 이유 1위가 "죄책감 없이 마심"(○○조사, 2025);
      경쟁사 리뷰에서도 '맛있는데 부담 없다'가 반복(리뷰 분석).
시사점: 메시지를 '건강식품' 톤이 아니라 '즐겨도 되는 핑계' 톤으로.
        여름 캠페인 헤드라인 후보에 반영.
```

주장은 근거 없이는 쓰지 않는다. 근거를 못 찾으면 그것은 인사이트가 아니라 가설이다. **"가설"이라고 표기**한다.

### 4단계 — 우선순위 매기기

인사이트를 3~5개로 압축한다. 많으면 힘이 빠진다. 판단 기준은 두 가지: **실행 임팩트(크게 바꾸나)** 와 **근거 강도(얼마나 확실한가)**. 임팩트 크고 근거 강한 것을 맨 위로 올린다.

## 산출물 형식

```markdown
# [주제] 인사이트

## 핵심 인사이트 (우선순위순)

### 인사이트 1 — [한 줄 주장]
- 근거: [사실] (출처, 시점) / [사실] (출처)
- 시사점: [실행 함의]
- 신뢰도: 강(교차검증됨) / 약(단일 출처) / 가설(근거 부족)

### 인사이트 2 ...

## 상충·미해결
- [항목]: A(출처) vs B(출처) — 판단 유보 이유

## 한 줄 결론
[전체를 관통하는 전략 방향 한 문장]
```

## 품질 기준

- [ ] 인사이트 3~5개로 압축, 우선순위 있음
- [ ] 모든 주장에 근거 + 출처
- [ ] 모든 인사이트에 시사점(so-what) 존재
- [ ] 각 인사이트에 신뢰도 표시(강/약/가설)
- [ ] 교차검증 흔적이 보임(여러 출처 대조)
- [ ] 상충·미해결은 숨기지 않고 명시
- [ ] 뻔한 말("건강 중시")에서 멈추지 않음

## 자주 하는 실수

- **요약을 인사이트로 착각** → 조사를 이어 붙이기만 하면 요약이다. 반드시 시사점까지.
- **주장만, 근거 없음** → 설득이 안 된다. 근거 못 대면 "가설"로 낮춘다.
- **한 자료만 보고 결론** → 교차검증 생략. 단일 출처는 표기하고 조심.
- **뻔한 결론** → "소비자는 X를 원한다" 수준이면 한 겹 더 판다. 남들이 놓친 각도를 찾는다.
- **상충을 임의로 봉합** → 억지로 하나로 만들지 말고 미해결로 남긴다.
- **너무 많은 인사이트** → 10개면 0개와 같다. 3~5개로.

## 재호출 지침

- "인사이트 다시" → 같은 조사로 각도를 바꿔 재도출한다.
- "더 날카롭게" → 뻔한 인사이트를 빼고 남들이 놓친 신호를 판다.
- "이 주장 근거 수정/보완" → 해당 인사이트의 근거만 교체·강화한다.
- "반대 근거도 검토" → 각 주장에 반증이 될 자료를 찾아 신뢰도를 재조정한다.

> AI는 매번 조금 다르게 종합한다. 도출된 인사이트를 그대로 쓰지 말고, 최소한 "근거가 진짜 주장을 받치는지" 한 번은 사람이 되짚는다.
