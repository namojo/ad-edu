# 비주얼 컨셉 생성 (Visual Concepting)

말로 된 컨셉을 **눈에 보이는 시안**으로 바꾸는 일. 키비주얼·무드보드·썸네일 컨셉을 여러 안으로 뽑아 고를 수 있게 한다.

## 왜 이 매뉴얼인가

컨셉은 말로만 하면 사람마다 다르게 상상한다. 시안을 여러 장 만들어 놓아야 방향을 빨리 좁힐 수 있다. 이 일이 실패하는 이유는 (1) 브리프를 그대로 프롬프트에 넣어 밋밋하거나, (2) 이미지 안에 글자를 넣어 오탈자가 나거나, (3) 만들고 파일 확인을 안 해 깨진 이미지를 넘기는 것이다. 이 매뉴얼은 **브리프→프롬프트 변환 → 병렬 생성 → 파일 검증**으로 이걸 막는다. 시안은 완성작이 아니라 **선택지**다. 여러 장을 빠르게 만드는 게 핵심이라 병렬 생성 도구(codex-image)를 쓴다.

## 시작 전 확보할 것

- **컨셉 브리프**: 무엇을 표현하나(제품/캠페인/무드), 핵심 메시지, 분위기 키워드
- **아트 디렉션 고정값**: 16:9 비율, 플랫 에디토리얼 벡터 스타일, 지정 색 팔레트, **이미지 안 텍스트 배제**
- **몇 안 뽑을지**: 보통 3~5안(병렬 한 번에 최대 5장)

## 워크플로우

### 1단계 — 컨셉 브리프 → 이미지 프롬프트 변환

브리프의 추상적 말을 이미지 도구가 알아듣는 **구체적 시각 묘사**로 바꾼다. 이게 이 매뉴얼의 핵심 기술이다.

브리프에서 뽑아 프롬프트에 담을 것:
- **주제·소재**: 무엇이 화면에 있나 (제품, 인물, 오브제, 장면)
- **분위기·톤**: 밝은/청량한/미니멀/역동적 등
- **스타일**: 플랫 벡터 / 에디토리얼 / 사진풍 등 (아트 디렉션 고정값 반영)
- **색**: 지정 팔레트를 색 이름·느낌으로 풀어서
- **구도·비율**: 16:9, 중앙/여백/클로즈업 등

**중요 — 이미지 안에 글자를 넣지 않는다.** 이미지 생성 도구는 글자를 자주 틀리게 그린다(오탈자). 카피는 나중에 디자인 단계에서 얹는다. 프롬프트에 "글자·텍스트·로고 없이"를 명시한다.

여러 안을 만들 때는 **한 축씩 다르게** 변주한다. (예: 안1은 청량, 안2는 역동, 안3은 미니멀.) 그래야 비교가 의미 있다.

### 2단계 — codex-image 로 병렬 생성

이미지는 `codex-image` 스킬(병렬 이미지 생성 도구)로 만든다. 한 번에 최대 5장을 동시에 만들 수 있어 시안 뽑기에 최적이다.

**핵심 규칙:**
- 각 이미지의 **출력 파일명을 모두 다르게** 한다. 같은 이름이면 마지막 것만 남는다.
- 파일명은 내용을 알아볼 수 있게. (예: `kv_concept_01_cool.png`)
- 최대 5장씩 배치. 6장 이상이면 5+나머지로 나눠 순차 배치한다.
- 백그라운드로 동시에 띄운다. 완료는 자동 통지되니 `sleep`으로 기다리지 않는다.

**실제 명령 (한 장):**
```bash
CODEX_HOME="/Users/andy/.codex" codex exec --sandbox workspace-write --skip-git-repo-check --cd <이미지_저장_폴더> -o /tmp/x.md "이미지 생성 도구로 '<이미지 프롬프트>' 를 생성하고 ./<파일>.png 로 저장. 경로만 보고."
```

**5안을 병렬로 만들 때** — 위 명령을 파일명·프롬프트만 바꿔 5개를 각각 백그라운드로 동시에 실행한다. 예:
```bash
# 안1
CODEX_HOME="/Users/andy/.codex" codex exec --sandbox workspace-write --skip-git-repo-check --cd ./images -o /tmp/kv1.md "이미지 생성 도구로 '청량한 여름 무드, 플랫 에디토리얼 벡터, 민트·코랄 팔레트, 16:9, 글자 없이' 를 생성하고 ./kv_01_cool.png 로 저장. 경로만 보고."
# 안2 (파일명·프롬프트만 바꿔 동일 패턴, 백그라운드 동시 실행)
CODEX_HOME="/Users/andy/.codex" codex exec --sandbox workspace-write --skip-git-repo-check --cd ./images -o /tmp/kv2.md "이미지 생성 도구로 '역동적인 움직임, ... 16:9, 글자 없이' 를 생성하고 ./kv_02_dynamic.png 로 저장. 경로만 보고."
```

> codex 실행 경로나 로그인 문제가 나면 `codex-image` 스킬 본문의 사전 점검·트러블슈팅을 따른다. (`codex` 대신 `/Applications/Codex.app/Contents/Resources/codex` 직접 실행이 필요할 수 있음.)

### 3단계 — 생성 후 파일 검증 (필수)

만들었다는 보고만 믿지 말고 **실제 파일을 확인**한다. AI 이미지는 0바이트로 깨지거나 폴더에 복사가 안 되는 경우가 있다.

```bash
file ./images/*.png      # 진짜 PNG 이미지인지 확인
ls -la ./images/*.png    # 0바이트 아닌지, 5개 다 있는지 확인
```

- 0바이트·손상 파일이 나오면 **그 안만 재생성**한다.
- 파일이 저장 폴더에 없고 codex 기본 폴더에만 있으면, 프롬프트의 "./<파일>로 저장"을 강화해 다시 만든다.
- 가능하면 실제 이미지를 열어 육안으로 컨셉·비율·글자없음을 확인한다.

### 4단계 — 시안 정리해서 넘기기

검증된 이미지들을 안별로 정리한다. 각 안에 **어떤 컨셉인지 한 줄 캡션**을 단다. (예: "안1 — 청량 무드, 여름 소구.") 넘기면 검수 역할(creative-direction)이 브랜드 기준으로 심사한다.

## 산출물 형식

```markdown
# [캠페인] 비주얼 컨셉 — [키비주얼/무드보드/썸네일]
> 아트 디렉션: 16:9 / 플랫 에디토리얼 벡터 / [팔레트] / 이미지 내 텍스트 없음

## 안1 — [컨셉 한 줄]
![안1](./images/kv_01_cool.png)
- 프롬프트 요지: [ ]

## 안2 — [컨셉 한 줄]
...

## 검증 결과
- 파일 5개 모두 정상 PNG 확인 / [재생성한 안 있으면 표기]
```

## 품질 기준

- [ ] 브리프를 구체적 시각 묘사로 변환(밋밋한 복붙 아님)
- [ ] 각 안이 한 축씩 다르게 변주됨(비교 의미 있음)
- [ ] 16:9, 지정 팔레트, 플랫 벡터 등 아트 디렉션 준수
- [ ] 이미지 안에 글자 없음(프롬프트에 명시)
- [ ] 출력 파일명 모두 다름
- [ ] 생성 후 `file`/`ls`로 파일 검증(0바이트·누락 없음)
- [ ] 각 안에 컨셉 캡션

## 자주 하는 실수

- **브리프를 그대로 프롬프트에** → 밋밋하다. 시각 요소로 풀어 쓴다.
- **이미지 안에 글자 넣기** → 오탈자 확정. 반드시 "글자 없이", 카피는 나중에 얹는다.
- **파일명 겹침** → 5개를 만들었는데 1개만 남는다. 이름을 모두 다르게.
- **검증 생략** → 깨진 이미지를 넘긴다. `file`/`ls` 필수, 육안 확인 권장.
- **순차로 5장** → 느리다. 병렬 백그라운드로 동시에.
- **`sleep`으로 대기** → 완료는 자동 통지된다. 폴링 금지.
- **모든 안이 비슷** → 선택지가 안 된다. 축을 나눠 변주.

## 재호출 지침

- "비주얼 다시" → 같은 브리프로 새 변주를 병렬 생성한다.
- "컨셉 수정해서 다시" → 브리프를 반영해 프롬프트를 고쳐 재생성한다.
- "한 안만 보완" → 그 안의 파일명 하나만 바꿔 그 안만 재생성한다.
- "톤 바꿔서" → 팔레트·분위기 키워드를 바꿔 전체를 다시 만든다.

> AI 이미지는 매번 다르게 나온다. 마음에 드는 게 안 나오면 프롬프트를 다듬어 여러 번 돌린다. 그리고 넘기기 전, 파일이 진짜 열리는지와 글자 오류가 없는지 한 번은 반드시 의심한다.
