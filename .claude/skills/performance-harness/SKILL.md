# Performance Harness — 퍼포먼스 마케팅 하네스 (전문가 풀)

운영 중인 채널의 성과를 **필요한 채널 전문가만 골라 불러(전문가 풀) 진단하고, 인사이트 에디터가 하나의 월간 리포트로 통합(팬인)** 하는 오케스트레이터(팀장)입니다. 핵심 개념은 **"모든 팀원을 매번 부르지 않는다"** — 이번 달 돌린 채널(검색·소셜·커머스 등)만 진단하고, 안 돌린 채널 전문가는 부르지 않습니다.

**실행 모드:** 에이전트 팀(기본). 채널 진단은 서로 독립이므로 병렬로 돌리고, 통합·문서화는 순차로 잇습니다. 모든 에이전트는 `model: opus`로 호출합니다.

## 팀 구성 — 전문가 풀에서 채널만큼만 선택

이번 달 운영한 채널 수만큼 performance-marketer를 **채널별로** 호출합니다. 아래 풀에서 필요한 팀원만 고릅니다(전부 쓰지 않는다).

| 후보 팀원(에이전트) | 언제 배정 | 사용 매뉴얼(스킬) | 출력 |
|---|---|---|---|
| performance-marketer(검색) | 검색광고(SEM/SEO)를 돌렸을 때 | performance-marketing | `_workspace/1_performance-marketer_search.md` |
| performance-marketer(소셜) | 인스타·틱톡 소셜을 돌렸을 때 | performance-marketing | `_workspace/1_performance-marketer_social.md` |
| performance-marketer(커머스) | 스마트스토어 등 커머스를 돌렸을 때 | performance-marketing | `_workspace/1_performance-marketer_commerce.md` |
| performance-marketer(디스플레이 등) | 해당 채널을 돌렸을 때만 | performance-marketing | `_workspace/1_performance-marketer_{채널}.md` |
| insight-synthesizer | 채널 카드 통합·예산 재배분·리포트(팬인) | insight-synthesis | `_workspace/2_insight_report.md` |
| doc-producer | KPI 시트(Excel)·발표 문서가 필요할 때만 | doc-automation (xlsx/pptx) | `_workspace/3_doc-producer_*` |

> 이번 실습의 기본 조합: performance-marketer 3명(검색·소셜·커머스) + insight-synthesizer 1명(통합) + (선택) doc-producer(KPI 시트). 디스플레이·유튜브를 안 돌렸다면 그 전문가는 부르지 않는다 — 이것이 전문가 풀의 핵심이다.

## Phase 0: 컨텍스트 확인 (초기/후속/부분 재실행 판별)

1. `_workspace/` 존재 여부를 확인한다.
2. 실행 모드 결정:
   - **없음** → 초기 실행. Phase 1로.
   - **있음 + 특정 채널만 수정 요청**(예: "소셜만 다시 진단", "커머스 보완") → 부분 재실행. 해당 채널 performance-marketer 1명만 재호출해 그 카드만 덮어쓰고, 이어서 insight-synthesizer만 재실행해 리포트를 갱신한다.
   - **있음 + 새 기간(이번 달) 데이터** → 정기 재실행. 기존 `_workspace/`를 `_workspace_{YYYYMM}/`로 보관하고 새 데이터로 재실행하되, **하네스 구조(팀·리포트 템플릿)는 재사용**한다.
3. 부분 재실행 시 나머지 채널 카드는 보존해 통합에 재사용한다.

## Phase 흐름 (전문가 선택 → 채널 병렬 진단 → 통합 → 문서화)

### Phase 1: 준비 — 채널·목표 고정
1. 브리프에서 **브랜드·기간·총광고비·운영 채널·채널별 목표(ROAS·CPA 등)** 를 고정한다.
2. 이번 달 **운영한 채널만** 추린다(안 돌린 채널은 진단 대상이 아니다).
3. 채널 성과 데이터를 `_workspace/00_input/`에 저장한다. 목표가 없으면 카테고리 통상 기준을 "가정" 라벨로 세운다.

### Phase 2: 팀 구성
1. `performance-team`을 만들고, 추린 채널 수만큼 performance-marketer + insight-synthesizer 1명을 `model: opus`로 등록한다.
2. 작업 등록: 채널별 진단 작업(각 1개)은 서로 의존이 없다. 통합 작업 1개는 모든 채널 진단 작업에 `depends_on`으로 연결한다.

### Phase 3: 채널 병렬 진단 (전문가 풀)
**실행 방식:** 팀원 자체 조율. 채널 마케터들이 공유 작업 목록에서 각자의 채널 진단 작업을 요청(claim)해 **동시에** 진행한다.
- 각 마케터는 **배정된 한 채널만** 판다(채널을 섞으면 승부처가 뭉개진다).
- KPI를 목표와 대비해 **판정(초과/이내/미달)** 하고, 원인은 **가설**로 표시, 카드를 "주장→근거→액션" 구조로 쓴다.
- 채널별 결과를 `_workspace/1_performance-marketer_{채널}.md`로 저장한다.

### Phase 4: 통합·예산 재배분 (팬인)
1. insight-synthesizer가 채널 카드를 모두 Read한다.
2. 채널 간 비교: 효율(ROAS·CPA)과 역할(인지 vs 전환)을 함께 놓고 **예산 재배분표**를 만든다. 효율 좋은 채널로 옮기되 인지 채널은 실험과 함께 유지.
3. 채널 카드의 원인 가설을 모아 **A/B 개선 실험 백로그**(가설·변인 하나·성공지표·판단기준)로 정리한다.
4. **검산:** 재배분 채널 금액 합 = 총예산, 비중 합 = 100%인지 확인해 리포트에 명시한다.
5. 결과를 `_workspace/2_insight_report.md`(월간 퍼포먼스 리포트)로 저장한다.

### Phase 5: 문서화·마무리
1. (선택) KPI 시트(Excel)나 발표 문서가 필요하면 doc-producer를 호출해 재배분표·KPI를 xlsx/pptx로 산출한다(doc-automation). 필요 없으면 부르지 않는다.
2. 최종 리포트를 지정 경로 `course5-performance/사례/C5_퍼포먼스_월간리포트.md`로 내보낸다.
3. 팀 정리, `_workspace/` 보존.

## 데이터 전달

- **팀 모드:** 태스크 + 파일 + 메시지. 통합 담당(insight-synthesizer)이 채널 카드 경로를 받아 Read로 수집한다.
- **파일 규약:** `_workspace/{phase}_{agent}_{artifact}.{ext}`. 채널 구분을 위해 artifact 자리에 채널명을 쓴다(search/social/commerce).
- **최종 산출물만** `course5-performance/사례/…`로 내보낸다. Excel/PPT가 필요하면 doc-producer의 파일을 함께 회수한다.

## 데이터 흐름(그림)

```
                    ┌─▶ [performance-marketer 검색] ─▶ 1_..._search.md ───┐
[팀장] → 채널 선택 ─┼─▶ [performance-marketer 소셜] ─▶ 1_..._social.md ───┤
 (전문가 풀, 동시)   └─▶ [performance-marketer 커머스]─▶ 1_..._commerce.md ─┤
   ※ 안 돌린 채널 전문가는 부르지 않음               (팬인) Read ◀──────────┘
                                                        ▼
                          [insight-synthesizer] 통합·예산 재배분·실험 백로그
                                                        ▼
                          (선택) [doc-producer] KPI 시트(Excel)·발표 문서
                                                        ▼
                          course5-performance/사례/C5_퍼포먼스_월간리포트.md
```

> **비결정성 안내:** 같은 데이터라도 매 실행마다 진단 표현이나 액션 우선순위가 조금씩 달라질 수 있습니다. 팀 구성과 리포트 뼈대(채널 카드 → 재배분 → 실험 백로그)가 위와 같으면 정상입니다.
> **검증 습관:** 예산 재배분표는 반드시 합계를 검산하고, 성과 수치와 원인 가설은 그대로 믿지 말고 **한 번은 의심하고** 확인한 뒤 확정하세요.

## 에러 핸들링

| 상황 | 대응 |
|---|---|
| 채널 마케터 1명 진단 실패 | 방식 바꿔 1회 재시도 → 실패 시 해당 채널 "진단 보류" 명시, 나머지 채널로 통합 진행 |
| 지표 데이터 일부 누락 | 지어내지 말고 있는 지표로 진단, "○○ 지표 미확보로 판정 보류" 명시 |
| 채널 간 수치 상충(광고 관리자 vs 정산) | 삭제 금지, 두 출처를 병기하고 판단 근거를 적음 |
| 예산 재배분 합 불일치 | 배분을 멈추고 어긋난 항목 표시 후 재조정, 합계·비중 검산 문구 필수 |
| 안 돌린 채널까지 진단 시도 | 운영 채널만 진단(전문가 풀 원칙), 나머지는 부르지 않음 |
| 근거 없는 '느낌' 액션 | 반드시 채널 카드의 수치 근거로 연결, 근거 없으면 액션에서 제외 |

## 팀 크기

**가변(3~5명).** 운영 채널 수에 따라 performance-marketer가 늘거나 준다. 채널 2~4명(운영한 것만) + insight-synthesizer 1(필수) + doc-producer(문서 필요 시)가 상한 가이드. 필요 없는 채널 전문가는 부르지 않는다(전문가 풀 원칙).

## 테스트 시나리오

### 정상 흐름
1. 사용자가 "제로톡" 런칭 4주차, 검색·소셜·커머스 3채널 성과 진단을 요청(총광고비 2,000만원, 채널별 목표 제공).
2. Phase 1에서 운영 채널 3개·목표 고정.
3. Phase 3에서 performance-marketer 3명이 검색/소셜/커머스를 병렬 진단 → 채널 카드 3개 저장.
4. Phase 4에서 insight-synthesizer가 통합·예산 재배분(합계 검산)·실험 백로그 도출.
5. 최종 리포트 `course5-performance/사례/C5_퍼포먼스_월간리포트.md` 생성.

### 에러 흐름
1. Phase 3에서 커머스 마케터가 스마트스토어 정산 매출을 못 받아 ROAS 계산 불가.
2. 1회 재시도(광고 관리자 매출로 대체) 후에도 확정 불가 → "커머스 ROAS 판정 보류(정산 데이터 미확보)"로 명시.
3. insight-synthesizer는 나머지 근거로 재배분하되, 커머스 배분에 "정산 확인 후 조정" 라벨.
4. 최종 리포트에 미확보 영역과 확인 방법을 명시.

## 팀장 완주 체크

- [ ] 운영한 채널만 골라 진단했는가(안 돌린 채널 전문가는 안 불렀는가)
- [ ] 채널 카드가 KPI 판정(초과/이내/미달)과 "주장→근거→액션" 구조인가
- [ ] 원인이 전부 "가설"로 표시됐는가
- [ ] 예산 재배분표의 합계·비중이 검산되었는가(합=총예산, 비중=100%)
- [ ] A/B 실험 백로그의 변인이 각각 하나인가
- [ ] 최종 리포트만 `course5-performance/사례/`로 내보냈는가
