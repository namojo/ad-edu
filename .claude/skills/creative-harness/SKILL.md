# Creative Harness — 크리에이티브 제작 하네스 (생성-검증)

카피와 키비주얼을 **생성**하는 팀원과, 브랜드 톤앤매너 기준으로 **검수·반려**하는 아트디렉터를 분리해 품질 게이트를 세우는 오케스트레이터(팀장)입니다. 만드는 사람과 검수하는 사람을 나누는 것(자기검증 금지)이 이 패턴의 핵심입니다.

**실행 모드:** 에이전트 팀(기본). 생성 2인(카피·비주얼)이 병렬로 만들고, 아트디렉터가 검수 게이트에서 반려하면 생성 팀원이 재생성하는 **생성-검증 루프**를 돌립니다. 모든 에이전트는 `model: opus`로 호출합니다.

## 팀 구성 (에이전트 풀에서)

| 팀원(에이전트) | 역할 | 사용 매뉴얼(스킬) | 출력 |
|---|---|---|---|
| copywriter | 감성/기능/유머 3앵글 카피 생성 | ad-copywriting | `_workspace/1_copywriter_copy.md` |
| visual-designer | 키비주얼 컨셉 4안 생성 | visual-concepting (+codex-image) | `_workspace/1_visual_keyvisual/` (PNG 4장) |
| art-director | 톤 기준 검수·반려/재생성 지시 | creative-direction | `_workspace/2_artdirector_review.md` |

## Phase 0: 컨텍스트 확인 (초기/후속/부분 재실행 판별)

1. `_workspace/` 존재 여부 확인.
2. 실행 모드 결정:
   - **없음** → 초기 실행. Phase 1로.
   - **있음 + 부분 수정 요청**(예: "카피만 다시", "비주얼 4안만 재생성", "톤 기준 강화 후 재검수") → 부분 재실행. 해당 생성 팀원만 재호출하고, 반드시 art-director 검수를 다시 통과시킨다(생성물이 바뀌면 재검수).
   - **있음 + 새 캠페인** → 새 실행. 기존 `_workspace/`를 타임스탬프 폴더로 이동 후 새로 시작.
3. 부분 재실행 시 이전 검수 코멘트(`2_artdirector_review.md`)를 생성 팀원 브리프에 넣어 반영시킨다.

## Phase 흐름 (생성 → 검증 → (반려 시) 재생성)

### Phase 1: 준비 — 검증 기준 먼저
1. 브리프에서 캠페인 목표와 **브랜드 톤앤매너 가이드**를 뽑아 `_workspace/00_input/tone_guide.md`로 명문화한다(기준이 없으면 검수가 무의미하다).
2. 카피 3앵글(감성/기능/유머)과 키비주얼 4안이라는 산출 목표를 확정한다.

### Phase 2: 팀 구성
1. `creative-team`을 만들고 copywriter, visual-designer, art-director를 `model: opus`로 등록.
2. 작업 등록: 카피 생성·비주얼 생성 2개는 병렬(의존 없음), 검수 작업 1개는 두 생성 작업에 `depends_on`으로 연결.

### Phase 3: 생성 (병렬)
1. copywriter: 감성/기능/유머 3앵글 × 변형을 `_workspace/1_copywriter_copy.md`로 저장.
2. visual-designer: **codex-image로 4안을 병렬 생성**(16:9, 통일 팔레트, **이미지 내 텍스트 배제**). 파일명은 유일하게 `_workspace/1_visual_keyvisual/kv-01~04.png`. 생성 후 0바이트·손상 여부를 확인하고 실패분만 재생성.

### Phase 4: 검증 (아트디렉터 게이트)
1. art-director가 `tone_guide.md` 기준으로 카피·비주얼을 검수한다.
2. 항목별로 **통과 / 반려(사유+수정 지시)** 를 `_workspace/2_artdirector_review.md`에 적는다.
3. **반려가 있으면** 해당 생성 팀원에게 SendMessage로 수정 지시를 보내 재생성 → 다시 Phase 4 검수(생성-검증 루프). 통과할 때까지 반복하되 루프는 최대 2회로 제한한다.

### Phase 5: 마무리
1. 최종 패키지를 정리: **추천 카피 3세트 + 키비주얼 4안 + 아트디렉터 코멘트 + 무드보드**.
2. 최종 산출물을 지정 경로 `course3-creative/사례/`(카피·코멘트 md)와 `course3-creative/images/`(선별 키비주얼)로 내보낸다.
3. 팀 정리, `_workspace/` 보존.

## 데이터 전달

- **팀 모드:** 태스크 + 파일 + 메시지. 반려/재생성 지시는 art-director → 생성 팀원 SendMessage로 전달하고, 근거는 검수 파일에 남긴다.
- **파일 규약:** `_workspace/{phase}_{agent}_{artifact}.{ext}`. 이미지는 폴더 `_workspace/1_visual_keyvisual/`에 유일 파일명으로.
- **최종 산출물만** `course3-creative/{사례,images}/`로 내보낸다.

## 데이터 흐름(그림)

```
[팀장] → 팀 구성 ─┬─▶ [copywriter] ──▶ 1_copywriter_copy.md ──┐
   (생성, 병렬)    └─▶ [visual-designer]▶ 1_visual_keyvisual/*  ┤
                                                         Read ◀─┘
                                                            ▼
                          [art-director] 검수 ── 통과? ──▶ 최종 패키지
                                              │
                                          반려(사유)
                                              ▼ SendMessage
                                   생성 팀원 재생성 → 재검수(최대 2회)
                                              ▼
                     course3-creative/{사례,images}/
```

> **비결정성 안내:** 카피와 이미지는 매번 다르게 나옵니다. 그래서 "만드는 사람"과 "검수하는 사람"을 나누고, 반려·재생성 루프로 기준에 맞을 때까지 좁혀갑니다.
> **검증 습관:** 첫 결과에 만족해 검수 게이트를 건너뛰지 마세요. 최소 1회는 아트디렉터 검수를 통과시키는 것이 이 패턴의 핵심 가치입니다.

## 에러 핸들링

| 상황 | 대응 |
|---|---|
| 톤 가이드 부재로 검수 불가 | Phase 1로 되돌려 기준을 먼저 명문화 |
| 이미지 0바이트/손상 | 해당 컷만 1회 재생성, 그래도 실패 시 3안으로 진행하고 누락 명시 |
| 첫 결과에 만족해 루프 생략 | 최소 1회는 검수 게이트를 거치도록 강제(생성-검증 가치) |
| 반려 사유가 모호 | 아트디렉터에게 "무엇을·왜·어떻게"를 명시하게 재작성 요구 |
| 카피/비주얼 관점 상충 | 어느 하나를 지우지 말고 아트디렉터가 근거와 함께 우선순위 판단 |

## 팀 크기

**중규모(3명).** 생성 2 + 검증 1이 이 패턴의 최소 구성이다. 앵글/포맷이 많아지면 카피라이터를 추가할 수 있으나 검수(art-director)는 1명으로 유지해 기준을 일관되게 지킨다.

## 테스트 시나리오

### 정상 흐름
1. 사용자가 "제로톡" 여름 캠페인 브리프와 톤 가이드를 입력.
2. Phase 1에서 톤 가이드를 명문화.
3. Phase 3에서 copywriter가 3앵글 카피, visual-designer가 codex-image로 4안 생성.
4. Phase 4에서 art-director가 검수 → 1안 톤 미달 반려 → 재생성 후 통과.
5. 최종 패키지를 `course3-creative/사례/`, `course3-creative/images/`로 내보냄.

### 에러 흐름
1. Phase 3에서 키비주얼 4안 중 1장이 0바이트로 생성.
2. 해당 컷만 1회 재생성 시도.
3. 재실패 시 3안으로 진행하고 "키비주얼 1안 미생성"을 명시.
4. art-director는 확보된 3안만 검수, 최종 보고에 누락 표기.

## 팀장 완주 체크

- [ ] 검증 기준(톤 가이드)을 Phase 1에서 먼저 명문화했는가
- [ ] 카피(3앵글)와 키비주얼(4안)이 병렬로 생성됐는가
- [ ] 이미지에 텍스트를 넣지 않았고 손상 파일을 걸러냈는가
- [ ] 아트디렉터 검수 게이트를 최소 1회 통과시켰는가(반려→재생성 확인)
- [ ] 최종 패키지만 `course3-creative/{사례,images}/`로 내보냈는가

## 실습 트리거 대표 프롬프트

```text
크리에이티브 제작 하네스를 구성해줘. "제로톡" 여름 캠페인용으로, 광고 카피를 감성/기능/유머 세 앵글로 뽑는 카피라이터, 키비주얼 컨셉을 여러 장 생성하는 비주얼 디자이너(codex-image 사용), 그리고 브랜드 톤앤매너 기준으로 결과를 검수하고 반려/재생성을 지시하는 아트디렉터가 필요해. 최종적으로 '추천 카피 3 + 키비주얼 컨셉 4안 + 아트디렉터 코멘트'를 줘.
```
