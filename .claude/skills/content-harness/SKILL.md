# Content Harness — 콘텐츠·SNS 제작 하네스 (파이프라인 심화)

**트렌드 조사 → 콘텐츠 기획 → 게시물 카피·캡션 → 썸네일 컨셉 → 제목/해시태그 SEO** 를 순차로 잇는 파이프라인 오케스트레이터(팀장)입니다. 앞 단계의 산출물이 다음 단계의 입력이 되는 순차 의존이 이 패턴의 핵심입니다.

> 이 하네스는 **게시물 카피·캡션**(피드·캡션·숏폼 문구)을 산출합니다. 촬영·연출용 **영상 대본(시나리오)·스토리보드·샷 프롬프트**가 필요하면 `video-content-harness`(선택 학습 E1)로 넘깁니다.

**실행 모드:** 에이전트 팀(기본). 단계 간 데이터 전달이 명확해야 하므로, 각 단계는 앞 단계 산출물 파일을 Read해 시작합니다. 모든 에이전트는 `model: opus`로 호출합니다.

## 팀 구성 (에이전트 풀에서)

| 팀원(에이전트) | 파이프라인 단계 | 사용 매뉴얼(스킬) | 출력 |
|---|---|---|---|
| researcher | 1) 최신 트렌드 조사 | market-research | `_workspace/1_researcher_trend.md` |
| content-creator | 2) 1주 콘텐츠 기획 + 게시물 카피·캡션 5세트 | content-production | `_workspace/2_content_plan_scripts.md` |
| visual-designer | 3) 콘텐츠별 썸네일 컨셉 | visual-concepting (+codex-image) | `_workspace/3_visual_thumbs/` (PNG) |
| content-creator | 4) 제목/해시태그 SEO 마무리 | content-production | `_workspace/4_content_seo.md` |

> 4단계 SEO 마무리는 content-creator가 다시 맡습니다(기획 맥락을 이미 아는 사람이 제목·해시태그를 최적화하는 것이 효율적).

## Phase 0: 컨텍스트 확인 (초기/후속/부분 재실행 판별)

1. `_workspace/` 존재 여부 확인.
2. 실행 모드 결정:
   - **없음** → 초기 실행. Phase 1로.
   - **있음 + 부분 수정 요청**(예: "카피·캡션만 다시", "썸네일만 재생성", "해시태그만 보완") → 부분 재실행. 해당 단계 팀원만 재호출하고 그 산출물만 덮어쓴다. **파이프라인 하류 단계는 상류가 바뀌면 함께 갱신**한다(카피·캡션이 바뀌면 썸네일·SEO도 재검토).
   - **있음 + 새 브랜드/주제** → 새 실행. 기존 `_workspace/`를 타임스탬프 폴더로 이동 후 새로 시작.
3. 부분 재실행 시 이전 단계 산출물 경로를 팀원 브리프에 넣는다.

## Phase 흐름 (파이프라인: 트렌드 → 기획·카피·캡션 → 썸네일 → SEO)

### Phase 1: 준비
1. 브리프에서 **브랜드, 플랫폼(숏폼/유튜브/인스타/블로그), 주제, 기간(1주)** 을 뽑아 `_workspace/00_input/brief.md`로 저장.
2. 파이프라인 4단계와 최종 산출물(캘린더·카피·캡션5·썸네일·해시태그)을 확정.

### Phase 2: 팀 구성
1. `content-team`을 만들고 researcher, content-creator, visual-designer를 `model: opus`로 등록.
2. 작업을 순차 의존으로 등록: 기획은 트렌드에, 썸네일은 기획·카피·캡션에, SEO는 카피·캡션에 각각 `depends_on`.

### Phase 3: 파이프라인 실행 (단계별 전달)
1. **트렌드(researcher):** 주제·플랫폼 관련 최신 트렌드를 `_workspace/1_researcher_trend.md`로 저장.
2. **기획·카피·캡션(content-creator):** 트렌드 파일을 Read → 1주 콘텐츠 5개 기획 + 각 게시물 카피·캡션(훅→본문→CTA)을 `_workspace/2_content_plan_scripts.md`로 저장. **플랫폼별 포맷 차이**를 반영한다.
3. **썸네일(visual-designer):** 기획·카피·캡션을 Read → codex-image로 콘텐츠별 썸네일 컨셉 생성(16:9, 통일 팔레트, **이미지 내 텍스트 배제** — 텍스트는 편집 단계에서 별도). `_workspace/3_visual_thumbs/thumb-01~05.png`, 생성 후 손상 확인·실패분 재생성.
4. **SEO(content-creator):** 카피·캡션을 Read → 콘텐츠별 제목·해시태그를 SEO 최적화해 `_workspace/4_content_seo.md`로 저장.
5. 각 단계 산출물을 다음 단계로 넘기기 전에 한 번씩 검토한다(중간 검토 생략 금지).

### Phase 4: 마무리 — 캘린더 통합
1. 4단계 산출물을 모아 **1주 콘텐츠 캘린더**(요일 × 콘텐츠 × 카피·캡션 × 썸네일 × 해시태그)로 정리.
2. 최종 산출물을 지정 경로로 내보낸다: 캘린더 `course5-content/사례/C5_제로톡_1주캘린더.md`, 썸네일 `course5-content/images/`.
3. 팀 정리, `_workspace/` 보존.

## 데이터 전달

- **팀 모드:** 태스크 + 파일 + 메시지. 각 단계는 **직전 단계의 산출물 파일 경로**를 받아 Read로 시작한다(파이프라인 전달의 핵심).
- **파일 규약:** `_workspace/{phase}_{agent}_{artifact}.{ext}`. 썸네일은 폴더에 유일 파일명으로.
- **최종 산출물만** `course5-content/{사례,images}/`로 내보낸다.

## 데이터 흐름(그림)

```
[팀장] → 팀 구성
   ▼
[researcher 트렌드] ─▶ 1_researcher_trend.md
   ▼ Read
[content-creator 기획·카피·캡션] ─▶ 2_content_plan_scripts.md
   ▼ Read
[visual-designer 썸네일] ─▶ 3_visual_thumbs/*.png
   ▼ Read(카피·캡션)
[content-creator SEO] ─▶ 4_content_seo.md
   ▼ 팀장이 통합
course5-content/{사례,images}/ (1주 캘린더 + 썸네일)
```

> **비결정성 안내:** 파이프라인은 앞 단계 결과가 조금만 바뀌어도 뒤 단계가 달라집니다. 그래서 상류(카피·캡션)를 고치면 하류(썸네일·SEO)를 함께 갱신합니다.
> **검증 습관:** 단계마다 산출물을 한 번씩 검토하고 넘기세요. 중간 검토를 생략하면 마지막 캘린더에서 오류가 몰립니다.

## 에러 핸들링

| 상황 | 대응 |
|---|---|
| 플랫폼 특성 무시한 동일 포맷 남발 | content-creator에게 플랫폼별 최적화를 명시적으로 재지시 |
| 썸네일에 텍스트 렌더 요구 | 텍스트 배제 원칙 적용, 텍스트는 편집 단계 분리 |
| 중간 산출물 검토 생략 | 단계마다 검토 게이트를 두고 통과 후 다음 단계 진행 |
| 트렌드 근거 부족 | researcher 1회 재시도 → 실패 시 "트렌드 근거 약함" 명시하고 진행 |
| 상류 수정 시 하류 미갱신 | 카피·캡션 변경 시 썸네일·SEO를 함께 재검토(파이프라인 일관성) |
| 영상 대본·시나리오·샷 프롬프트 요청 | 이 하네스 범위 밖 → video-content-harness(선택 학습 E1)로 안내 |

## 팀 크기

**중규모(3명, content-creator 2회 등판).** 트렌드·기획·카피·캡션·썸네일·SEO 4단계를 3명이 순차로 담당한다. 콘텐츠 수가 많아지면(예: 2주) 카피·캡션을 나눠 content-creator를 추가할 수 있다.

## 테스트 시나리오

### 정상 흐름
1. 사용자가 "제로톡" 1주 SNS 콘텐츠(숏폼/인스타/블로그)를 요청.
2. researcher가 트렌드 조사 → content-creator가 5개 기획·카피·캡션 → visual-designer가 썸네일 5안 → content-creator가 해시태그 SEO.
3. 1주 콘텐츠 캘린더로 통합, `course5-content/사례/C5_제로톡_1주캘린더.md` + 썸네일 내보냄.

### 에러 흐름
1. Phase 3-3에서 썸네일 1장이 손상 생성.
2. 해당 컷만 1회 재생성, 실패 시 "썸네일 1건 미생성" 명시.
3. content-creator는 확보된 카피·캡션·썸네일로 SEO·캘린더 완성.
4. 최종 보고에 누락 항목 표기.

## 팀장 완주 체크

- [ ] 파이프라인 4단계가 순서대로 실행되고 각 단계가 앞 파일을 읽었는가
- [ ] 콘텐츠가 플랫폼별로 최적화됐는가(동일 포맷 남발 아님)
- [ ] 썸네일에 텍스트를 넣지 않았고 손상 파일을 걸러냈는가
- [ ] 제목·해시태그 SEO가 카피·캡션 기반으로 마무리됐는가
- [ ] 1주 캘린더와 썸네일만 `course5-content/{사례,images}/`로 내보냈는가

## 실습 트리거 대표 프롬프트

```text
콘텐츠·SNS 제작 하네스를 구성해줘. "제로톡" 브랜드로, 최신 트렌드를 조사하고 그걸 바탕으로 한 주치 콘텐츠를 기획하고, 각 콘텐츠의 게시물 카피·캡션을 쓰고, 썸네일 컨셉을 생성(codex-image)하고, 마지막에 제목/해시태그를 SEO 최적화하는 순차 파이프라인 팀이 필요해. '1주 콘텐츠 캘린더 + 게시물 카피·캡션 5세트 + 썸네일 컨셉 + 해시태그 세트'를 줘.
```

후속 예: "월요일 콘텐츠 카피·캡션만 다시" · "썸네일 컨셉 5안 재생성" · "해시태그 SEO만 보완".
