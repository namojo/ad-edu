# 크리에이티브 디렉션 (검수)

카피와 비주얼이 **브랜드다운가**를 심사하는 관문. 만드는 사람과 심사하는 사람은 반드시 나뉘어야 품질이 오른다. 이 매뉴얼은 **심사자**의 업무다.

## 왜 이 매뉴얼인가

생성과 검증이 같은 손에서 이뤄지면 자기 결과를 후하게 본다. 그래서 크리에이티브는 항상 **만든 뒤 다른 눈으로 검수**한다(생성-검증 구조). 검수가 무너지는 이유는 (1) "느낌상 별로"처럼 기준 없이 까거나, (2) 반대로 다 통과시키거나, (3) 반려하면서 뭘 고치라는지 안 알려줘 무한 반복되는 것이다. 이 매뉴얼은 **기준 명문화 → 항목 판정 → 구체적 재생성 지시**로 이걸 막는다. 좋은 검수는 "이래서 안 되고, 이렇게 고치면 된다"가 나온다.

## 워크플로우

### 1단계 — 검수 기준 명문화 (먼저)

심사 전에 **무엇으로 심사할지**부터 글로 적는다. 기준 없는 검수는 취향 싸움이 된다. 브랜드 가이드가 있으면 그걸 기준으로 삼고, 없으면 아래 항목으로 기준표를 만들어 산출물에 남긴다.

| 검수 항목 | 무엇을 보나 |
|---|---|
| 톤앤매너 | 브랜드 성격(밝음/진중, 반말/존댓말)과 일치하는가 |
| 메시지 정합성 | 담기로 한 인사이트·핵심 강점이 살아 있는가 |
| 타깃 적합성 | 겨냥한 소비자에게 통하는가 |
| 표현 리스크 | 과장·허위·비교광고·민감·비하 소지가 없는가 |
| 완성도 | 오탈자, 어색함, 조악함이 없는가 |
| 브랜드 자산 | 로고·색·서체 등 시각 규칙(비주얼일 때)을 지켰는가 |

각 항목은 **통과/미달로 판정 가능한 문장**으로 적는다. ("세련되게" 같은 모호한 기준 대신 "브랜드 색 팔레트 밖의 색을 쓰지 않았는가".)

### 2단계 — 항목별 판정

산출물(카피 또는 이미지)을 항목마다 통과/미달로 매긴다. 근거를 반드시 한 줄 단다. 판정은 후하지도 박하지도 않게 — **기준에 비추어** 한다.

- **통과**: 기준 충족. 근거 한 줄.
- **미달**: 기준 위반. **무엇이 왜 미달인지 + 어떻게 고칠지**를 함께.

이미지는 육안으로 실제 파일을 열어 확인한다. "생성됐다"는 보고만 믿지 않는다. 특히 **이미지 안에 넣은 글자의 오탈자**, 브랜드 색 이탈, 비율 오류를 본다.

### 3단계 — 반려·재생성 지시

미달이 하나라도 있으면 **반려**한다. 반려는 비난이 아니라 다음 액션이다. 만든 역할이 바로 재작업할 수 있게 지시를 구체적으로 준다.

**나쁜 반려:** "톤이 별로예요. 다시."
**좋은 반려:**
```
[반려] 헤드라인 3안
- 미달 항목: 톤앤매너
- 사유: 브랜드는 존댓말·진중 톤인데 유행어 반말("~각")을 씀
- 재생성 지시: 같은 메시지를 존댓말 단정형으로. 유행어 제거.
- 유지할 것: 기능 강점을 앞세운 구성은 좋음 — 살릴 것
```

살릴 부분("유지할 것")을 알려주면 재생성이 엉뚱한 방향으로 튀지 않는다.

### 4단계 — 재검수 (재생성본 확인)

되돌려받은 재생성본은 **지적한 항목이 실제로 고쳐졌는지** 다시 본다. 통과할 때까지 반복하되, 매번 같은 기준을 적용한다(기준을 슬그머니 낮추지 않는다).

> 재작업이 계속 겉돌면, 기준 자체가 모호하거나 인풋(브리프)이 부족한 것이다. 기준을 더 구체화하거나 상위에 브리프 보완을 요청한다.

## 산출물 형식

```markdown
# 크리에이티브 검수 결과
> 대상: [카피/이미지 등] / 브랜드 톤 기준: [요약]

## 검수 기준 (명문화)
- [항목]: [통과 조건]

## 항목별 판정
| 항목 | 판정 | 근거 |
|---|---|---|
| 톤앤매너 | 통과/미달 | ... |

## 종합 판정: 통과 / 반려

## 반려 시 — 재생성 지시
- 대상: [ ]
- 미달 항목·사유: [ ]
- 수정 지시: [ ]
- 유지할 것: [ ]
```

## 품질 기준

- [ ] 검수 전에 기준을 글로 명문화
- [ ] 모든 항목을 통과/미달로 판정 + 근거
- [ ] 이미지는 실제 파일을 열어 확인(보고만 믿지 않음)
- [ ] 반려 시 사유 + 구체적 수정 지시 + 유지할 것
- [ ] 재검수에서 같은 기준 적용(기준 안 낮춤)
- [ ] 생성과 검증이 분리됨(자기 결과 자가승인 아님)

## 자주 하는 실수

- **기준 없이 감으로 판정** → 취향 싸움. 1단계에서 기준을 적는다.
- **다 통과시키기** → 검수가 형식이 됨. 미달은 미달로.
- **반려만 하고 지시 없음** → 무한 재작업. "이렇게 고쳐라"까지.
- **보고만 믿고 이미지 안 봄** → 오탈자·색 이탈을 놓친다. 반드시 육안.
- **재검수에서 기준 완화** → 통과시키려고 눈감으면 검수가 무의미.
- **살릴 부분 안 알려줌** → 재생성이 엉뚱하게 튄다.

## 재호출 지침

- "다시 검수" → 같은 기준으로 재판정한다.
- "재생성본 재검수" → 지적 항목이 고쳐졌는지 중심으로 본다.
- "기준 보완" → 검수 기준표에 항목·조건을 추가·구체화한다.
- "이 항목만" → 해당 항목만 재판정하고 나머지 판정은 유지한다.

> AI가 만든 카피·이미지는 그럴듯해 보여도 톤이 어긋나거나 오탈자가 숨어 있다. 이 검수가 바로 "그냥 믿지 말고 한 번 의심하라"를 실행하는 자리다. 통과 도장은 신중히 찍는다.
