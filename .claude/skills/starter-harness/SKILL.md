# Starter Harness — 첫 하네스 (파이프라인 맛보기)

신제품 런칭을 돕는 **리서처 → 카피라이터** 2인 파이프라인을 조율하여, 시장 요약 1페이지와 광고 헤드라인 5개를 만드는 입문용 오케스트레이터(팀장)입니다. 앞사람(리서처)의 조사 결과를 뒷사람(카피라이터)이 받아 쓰는 순차 협업을 몸으로 익히게 합니다.

**실행 모드:** 에이전트 팀(기본). 단 이 과정은 팀원이 2명뿐인 소규모라, 팀 통신 기능이 없는 환경이거나 빠른 시연이 필요하면 **서브 에이전트 모드**로도 실행할 수 있습니다(아래 대안 표시). 두 모드 모두 모든 에이전트는 `model: opus`로 호출합니다.

## 팀 구성 (에이전트 풀에서)

| 팀원(에이전트) | 역할 | 사용 매뉴얼(스킬) | 출력 |
|---|---|---|---|
| researcher | 시장·경쟁을 빠르게 조사 | market-research | `_workspace/1_researcher_market.md` |
| copywriter | 조사 기반 헤드라인 5개 작성 | ad-copywriting | `_workspace/2_copywriter_headlines.md` |

## Phase 0: 컨텍스트 확인 (초기/후속/부분 재실행 판별)

1. 작업 폴더에 `_workspace/` 가 있는지 확인한다.
2. 실행 모드를 정한다:
   - **`_workspace/` 없음** → 초기 실행. Phase 1로 진행.
   - **`_workspace/` 있음 + 부분 수정 요청**(예: "헤드라인만 더 위트있게", "조사만 다시") → 부분 재실행. 해당 팀원만 다시 호출하고, 그 팀원의 산출물 파일만 덮어쓴다. 나머지는 보존한다.
   - **`_workspace/` 있음 + 새 브랜드/새 브리프** → 새 실행. 기존 `_workspace/`를 `_workspace_{YYYYMMDD_HHMMSS}/`로 옮긴 뒤 새로 시작한다.
3. 부분 재실행일 때는 이전 산출물 경로를 팀원 브리프에 넣어, 기존 결과를 읽고 피드백만 반영하도록 지시한다.

## Phase 흐름 (파이프라인: 조사 → 카피)

### Phase 1: 준비
1. 사용자 브리프에서 3요소를 뽑는다 — **도메인**(무슨 브랜드/카테고리), **역할**(리서처+카피라이터), **산출물**(요약 1p + 헤드라인 5개).
2. `_workspace/` 를 만들고 입력을 `_workspace/00_input/brief.md` 로 저장한다.

### Phase 2: 팀 구성
- 에이전트 팀 모드: `starter-team` 이라는 팀을 만들고 researcher, copywriter 2명을 `model: opus`로 등록한다. 작업 2개를 등록하되, 카피 작업은 조사 작업에 `depends_on`으로 연결한다(앞사람 결과가 있어야 뒷사람이 시작).
- 서브 에이전트 대안: 팀을 만들지 않고, 아래 Phase 3~4를 Agent 도구 순차 호출로 대체한다(리서처 먼저, 결과 파일을 읽어 카피라이터에 전달).

### Phase 3: 조사 (리서처)
1. researcher에게 브리프를 주고 시장·경쟁을 조사시킨다.
2. 결과를 `_workspace/1_researcher_market.md`(요약 → 근거·출처 → 공백 항목)로 저장하게 한다.
3. 리서처가 "확인 불가" 항목을 남겨도 정상이다. 억지로 채우지 않았는지 확인한다.

### Phase 4: 카피 (카피라이터)
1. copywriter에게 `_workspace/1_researcher_market.md`를 **입력으로 읽게** 한다(파이프라인 전달의 핵심).
2. 조사 근거에 기반한 헤드라인 5개를 `_workspace/2_copywriter_headlines.md`로 저장하게 한다.
3. 각 헤드라인이 어떤 조사 포인트에서 나왔는지 한 줄씩 근거를 붙이도록 지시한다.

### Phase 5: 마무리
1. 두 산출물을 읽어 최종본을 정리한다: **시장 요약 1페이지 + 헤드라인 5개**.
2. 최종 산출물만 지정 경로 `course1-basics/사례/C1_제로톡_미니결과.md` 로 내보낸다.
3. 이 흐름이 "파이프라인(조사→카피)"임을 사용자에게 라벨링해 안내한다.
4. 팀 모드면 팀을 정리하고, `_workspace/`는 검증용으로 보존한다.

## 데이터 전달

- **팀 모드:** 태스크(작업 지시) + 파일(산출물) + 메시지(SendMessage)로 조율한다. 카피라이터가 조사 결과가 필요하면 리서처 산출물 파일 경로를 메시지로 받아 Read한다.
- **파일 규약:** `_workspace/{phase}_{agent}_{artifact}.{ext}` (예: `_workspace/1_researcher_market.md`).
- **최종 산출물만** 지정 경로(`course1-basics/사례/…`)로 내보내고, 중간 파일은 `_workspace/`에 남긴다.

## 데이터 흐름(그림)

```
[팀장] → 팀 구성 → [researcher] ──파일──▶ 1_researcher_market.md
                                              │ (앞사람 결과)
                                              ▼ Read
                        [copywriter] ──파일──▶ 2_copywriter_headlines.md
                                              │
                                              ▼ 팀장이 통합
                              course1-basics/사례/C1_제로톡_미니결과.md
```

> **비결정성 안내:** AI는 같은 브리프에도 매번 조금씩 다르게 답합니다. 결과가 지난번과 달라도 정상이며, 마음에 드는 방향이 나올 때까지 톤·분량을 바꿔 재요청하면 됩니다(진화 체험).
> **검증 습관:** 결과를 그대로 믿지 말고 한 번은 의심하세요. 헤드라인이 조사 근거와 실제로 연결되는지 팀장이 최종 확인합니다.

## 에러 핸들링

| 상황 | 대응 |
|---|---|
| 리서처가 자료를 못 찾음 | 방식 바꿔 1회 재시도 → 그래도 없으면 "확인 불가"로 명시하고 진행 |
| 출처마다 값이 다름(상충) | 한쪽을 지우지 말고 두 값을 출처와 함께 병기 |
| 카피가 조사와 무관하게 나옴 | 조사 파일을 다시 읽게 하고 근거 연결을 지시, 1회 재생성 |
| 팀 기능 미활성화로 단일 응답 | 서브 에이전트 모드로 전환하거나 팀 기능 활성화 안내 |

## 팀 크기

**소규모(2명).** 입문 과정이므로 인원을 늘리지 않는다. 팀원은 researcher 1 + copywriter 1로 고정한다.

## 테스트 시나리오

### 정상 흐름
1. 사용자가 "제로톡" 미니 리서치+카피 브리프를 입력.
2. Phase 1에서 3요소(음료 런칭 / 리서처+카피 / 요약1p+헤드라인5) 도출.
3. 리서처가 시장 요약을 `1_researcher_market.md`로 저장.
4. 카피라이터가 그 파일을 읽고 헤드라인 5개를 `2_copywriter_headlines.md`로 저장.
5. 최종본 `course1-basics/사례/C1_제로톡_미니결과.md` 생성.
6. "더 위트있게" 재요청 시 Phase 0에서 부분 재실행으로 판별 → 카피라이터만 재호출.

### 에러 흐름
1. Phase 3에서 리서처가 경쟁사 데이터를 못 찾음.
2. 방식을 바꿔 1회 재시도, 그래도 없으면 "경쟁사 데이터 확인 불가"로 명시.
3. 카피라이터는 확보된 시장 요약만으로 헤드라인 생성.
4. 최종 보고에 "경쟁 데이터 일부 미확보" 를 명시.

## 팀장 완주 체크

- [ ] 브리프에서 3요소(도메인·역할·산출물)를 뽑았는가
- [ ] 리서처 산출물이 `_workspace/1_researcher_market.md`로 저장됐는가
- [ ] 카피라이터가 그 파일을 **읽고** 헤드라인을 뽑았는가(파이프라인 전달)
- [ ] 헤드라인마다 조사 근거가 한 줄씩 붙었는가
- [ ] 최종본만 `course1-basics/사례/`로 내보내고 `_workspace/`는 보존했는가
- [ ] 이 흐름이 "파이프라인(조사→카피)"임을 사용자에게 라벨링했는가
- [ ] 재요청("더 위트있게") 시 부분 재실행으로 카피만 다시 뽑았는가

## 실습 트리거 대표 프롬프트

```text
하네스 구성해줘. 신제품 음료 런칭을 돕는 작은 팀이 필요해 — 시장/경쟁 상황을 빠르게 조사하는 리서처와, 그 조사를 바탕으로 광고 헤드라인을 뽑는 카피라이터. 조사 요약 1페이지와 헤드라인 5개를 결과로 줘. (브랜드: 제로 슈거 스파클링 "제로톡")
```

후속 예: "헤드라인을 더 위트있게 다시 뽑아줘" · "조사만 최신 트렌드로 다시" · "이전 결과 기반으로 헤드라인 3개만 보완".
