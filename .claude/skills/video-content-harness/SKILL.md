# Video Content Harness — Seedance 영상 콘텐츠 하네스 (파이프라인)

> **선택 학습 E1.** 이 하네스는 커리큘럼의 **선택 과정(E1)** 입니다. 영상·시나리오(대본) 제작이 실제 업무인 실무자를 위한 심화 하네스로, 핵심 과정(C1~C8) 수료 후 필요할 때 사용합니다. 게시물 카피·캡션 중심의 SNS 콘텐츠는 `content-harness`(C6)가 담당합니다.

트렌드 조사에서 시작해 **대본 → 스토리보드/샷 설계 → Seedance 영상 생성 → 편집·배포 가이드**까지 순서대로 이어지는 오케스트레이터(팀장)다. 각 단계 산출물이 다음 단계의 입력이 되는 파이프라인이며, 최신 Seedance로 광고·숏폼 영상을 만든다.

**실행 모드:** 에이전트 팀(기본). 단계가 순차 의존이므로 앞 팀원의 파일이 완료돼야 다음 팀원이 시작한다. 모든 에이전트는 `model: opus`로 호출한다.

## 팀 구성 (에이전트 풀에서)

| 순서 | 팀원(에이전트) | 하는 일 | 사용 매뉴얼(스킬) | 출력 |
|---|---|---|---|---|
| 1 | researcher | 트렌드·레퍼런스 조사 | market-research | `_workspace/1_researcher_trend.md` |
| 2 | content-creator | 영상 대본(훅→전개→CTA) | content-production | `_workspace/2_content_script.md` |
| 3 | video-director | 스토리보드·샷별 Seedance 프롬프트 | seedance-video | `_workspace/3_video_storyboard.md` |
| 4 | visual-designer | (선택) 스토리보드 키프레임 스틸 | visual-concepting | `_workspace/3b_visual_keyframes/` |
| 5 | art-director | 브랜드 톤·상표 안전 검수 | creative-direction | `_workspace/4_review.md` |

> Seedance 실제 영상 렌더링은 외부 도구(Seedance 최신 버전)에서 수행한다. 이 하네스는 **"무엇을 어떻게 찍을지"(대본·스토리보드·프롬프트)와 편집·배포 설계**를 산출한다.

## Phase 0: 컨텍스트 확인 (초기/후속/부분 재실행)
1. `_workspace/` 존재 여부 확인.
2. 모드 결정:
   - 없음 → 초기 실행, Phase 1로.
   - 있음 + "특정 샷만 다시"·"훅 바꿔" → 부분 재실행. video-director만 재호출해 해당 샷 프롬프트를 고치고, 이후 검수만 다시.
   - 있음 + 새 캠페인 → 새 실행. 기존 `_workspace/`를 타임스탬프 폴더로 이동.

## Phase 흐름 (파이프라인)
1. **조사:** researcher가 플랫폼·주제·트렌드를 조사(숏폼 포맷·길이·후킹 패턴 포함).
2. **대본:** content-creator가 플랫폼별 대본을 훅→전개→CTA로 작성.
3. **샷 설계:** video-director가 대본을 3~6개 샷으로 분해하고, seedance-video 매뉴얼로 샷별 프롬프트를 작성(장면·동작·카메라·조명·색·오디오). 상표·문구는 넣지 않는다.
4. **(선택) 키프레임:** visual-designer가 스토리보드용 대표 스틸을 codex-image로 생성(이미지→영상 입력·프리뷰용).
5. **검수:** art-director가 브랜드 톤·상표 안전·훅 강도를 검수해 미달 샷을 반려.
6. **마무리:** 최종 스토리보드 + 영상 스펙(길이·해상도·비율·오디오) + 편집·자막·배포 가이드를 지정 경로로 내보낸다.

## 데이터 전달
- 팀 모드: 태스크 + 파일 + 메시지. 순차 파이프라인이라 각 팀원은 앞 단계 파일 경로를 받아 Read로 이어받는다.
- 파일 규약: `_workspace/{phase}_{agent}_{artifact}.{ext}`. 최종 산출물만 `elective1-video/사례/…`로 내보낸다.

## 데이터 흐름(그림)
```
[researcher] 트렌드 ─▶ [content-creator] 대본 ─▶ [video-director] 스토리보드+Seedance 프롬프트
                                                          │  ├─▶ [visual-designer] 키프레임(선택)
                                                          ▼
                                              [art-director] 톤·상표 검수 ─▶ 최종 스토리보드/배포 가이드
```
> **비결정성 안내:** Seedance 영상은 같은 프롬프트도 매번 조금씩 다르게 나옵니다. 재현 편차는 정상이며, 마음에 드는 샷의 프롬프트·설정을 기록해 재사용하세요.
> **검증 습관:** 생성된 샷을 반드시 육안 검수하고, 브랜드 톤·상표·어색한 움직임을 확인한 뒤 미달 샷만 다시 만드세요.

## 에러 핸들링
| 상황 | 대응 |
|---|---|
| 특정 샷 품질 미달 | 프롬프트 조정 1회 재시도 → 실패 시 대안 샷 제시, "미확보" 표시 |
| 상표·권리 위반 소지 | 생성 전 프롬프트에서 제거, art-director 검수 |
| 대본이 길어 샷 과다 | 핵심 메시지 1개로 좁혀 샷 수 축소 |
| 플랫폼 비율 혼동 | 릴스·쇼츠·틱톡 9:16 / 유튜브 가로 16:9로 분리 산출 |

## 팀 크기
**중규모(4~5명).** 파이프라인이라 동시 인원은 적지만 단계가 많다. 키프레임이 필요 없으면 visual-designer는 생략 가능.

## 테스트 시나리오
### 정상 흐름
1. "제로톡" 여름 숏폼 영상 캠페인을 요청.
2. researcher가 숏폼 트렌드 조사 → content-creator가 대본 3편 → video-director가 샷별 Seedance 프롬프트 → art-director 검수.
3. 최종 스토리보드+배포 가이드를 `elective1-video/사례/`에 생성.
### 에러 흐름
1. video-director의 한 샷이 상표 노출 소지.
2. art-director가 반려 → 해당 샷 프롬프트에서 상표 제거 후 재생성.
3. 최종본에 "상표 미포함" 확인을 명시.

## 팀장 완주 체크
- [ ] 대본이 훅(3초)→전개→CTA 구조인가
- [ ] 각 샷 프롬프트에 장면·동작·카메라·조명·색·오디오 6요소가 있는가
- [ ] 상표·문구가 프롬프트에 없는가(텍스트는 편집 단계)
- [ ] 플랫폼별 비율·길이가 맞는가
- [ ] 최종 스토리보드·배포 가이드만 지정 경로로 내보냈는가

## 실습 트리거 대표 프롬프트
```text
영상 하네스를 구성해줘. 제로 슈거 스파클링 음료 "제로톡" 여름 숏폼 캠페인 영상을 만들고 싶어. 트렌드 조사 → 대본 → 스토리보드와 Seedance(최신 버전) 샷 프롬프트 → 브랜드 톤 검수 순서로 진행하고, 릴스/쇼츠(9:16) 3편 분량의 스토리보드와 각 샷 프롬프트, 편집·자막·배포 가이드를 결과로 줘. 상표·문구는 영상에 넣지 말고 편집 단계로 남겨줘.
```
후속 예: "2편의 훅을 더 강하게 다시" · "가로 16:9 버전도 만들어줘" · "오디오를 더 잔잔하게".
