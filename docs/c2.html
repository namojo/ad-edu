1. 왜 이 하네스인가

실무 문제

대행사의 주간 캠페인 성과 리포트를 떠올려 보세요. 매주 월요일 아침, 담당자는 이런 일을 반복합니다.

  • 광고 매체 대시보드에서 숫자를 긁어 옵니다.
  • 엑셀에 붙여넣고 지난주와 비교표를 만듭니다.
  • "왜 클릭률이 떨어졌지?"를 고민해 코멘트를 답니다.
  • 다음 주에 뭘 바꿀지 액션을 정리합니다.
  • 이 모든 걸 발표용 덱과 보고용 문서로 각각 다시 옮깁니다.

같은 일을 매주, 브랜드마다 반복합니다. 손이 많이 가고, 사람이 바쁘면 코멘트가 부실해지고, 포맷도 그때그때 달라집니다.

해결

이 반복 업무를 한 번 설계해 두면, 다음 주부터는 데이터만 넣고 "지난주 리포트 만들어 줘" 한마디로 끝납니다. 팀장 격인 하네스가 필요한 팀원만 골라 일을 나눠 맡기고, 문서 담당 팀원이 덱·문서·시트·PDF까지 자동으로 마감합니다.

구분 Before (지금) After (하네스 도입 후)
소요 시간 브랜드당 2~3시간 데이터 입력 + 검토 20~30분
반복성 매주 처음부터 다시 설계 1회 → 매주 재실행
산출물 덱·문서 따로 수작업 4종(덱+문서+시트+PDF) 자동 생성
품질 담당자 컨디션에 좌우 매주 같은 틀·같은 항목 보장
개선 잘 안 쌓임 변경 이력으로 매주 진화

핵심은 "더 빨리"가 아니라 "한 번 잘 설계해 두고, 계속 재사용하며 다듬는다"입니다. 하네스는 한 번 쓰고 버리는 도구가 아니라, 쓸수록 똑똑해지는 우리 팀의 자산이 됩니다.

2. 개념 이해

이번 과정에는 새 개념이 다섯 개 등장합니다. 모두 광고팀 일하는 방식에 빗대어 이해하면 쉽습니다.

2-1. 계층적 위임 — "큰 일을 잘게 나눠 아래로 맡긴다"

팀장이 "이번 주 성과 리포트 만들어"라는 큰 일을 받으면, 혼자 다 하지 않습니다. 큰 일을 작은 일로 쪼개 담당자에게 나눠 맡깁니다.

주간 리포트 (큰 일)
├─ 데이터 요약   → 데이터 담당
├─ 인사이트 도출 → 전략 담당
└─ 액션 제안     → 기획 담당

각 담당자도 필요하면 자기 일을 더 잘게 나눠 아래로 맡길 수 있습니다. 이렇게 위에서 아래로 일이 재귀적으로 갈라지는 방식이 계층적 위임입니다. AE가 캠페인을 채널별로 나눠 각 채널 담당에게 맡기는 것과 똑같습니다.

2-2. 전문가 풀 — "상황에 맞는 사람만 골라 부른다"

우리 대행사에는 리서처, 카피라이터, 아트디렉터, 미디어 플래너, 콘텐츠 담당, 데이터 담당이 모두 있습니다. 하지만 매번 전원을 부르지는 않습니다. 리포트 업무면 데이터·전략·기획 담당만, 크리에이티브 업무면 카피·아트 담당만 부릅니다.

이렇게 상황에 맞는 팀원만 골라 호출하는 방식이 전문가 풀입니다. 필요 없는 사람을 부르지 않으니 빠르고, 결과도 깔끔합니다.

계층적 위임(일을 나눈다) + 전문가 풀(사람을 고른다) — 이 둘을 합치면, 팀장이 업무를 쪼갠 뒤 각 조각에 딱 맞는 전문가만 배치하는 유연한 팀이 됩니다. 이 자동화 하네스가 바로 이 조합입니다.

2-3. 문서 자동화 — "결과를 사람이 안 옮겨도 문서가 된다"

지금까지는 하네스가 만든 결과를 여러분이 직접 덱과 문서에 옮겨 담았습니다. 이제는 **문서 담당 팀원(문서 프로듀서)**이 그 일을 맡습니다. 하네스 결과를 받아 발표용 PPT, 상세 Word 보고서, 성과 Excel 시트, 통합 PDF로 알아서 마감합니다.

이때 문서 프로듀서는 우리가 이미 배운 **업무 매뉴얼(스킬)**을 씁니다 — PPT는 pptx, Word는 docx, Excel은 xlsx, PDF는 pdf 매뉴얼입니다. 여러분이 매뉴얼 이름을 외울 필요는 없습니다. "덱과 문서로 만들어 줘"라고 말하면 담당자가 알아서 적절한 매뉴얼을 꺼내 씁니다.

2-4. 하네스 진화 메커니즘 — "쓸수록 좋아지는 되먹임"

하네스는 한 번 만들고 끝이 아닙니다. 초기 → 사용 → 개선의 고리를 돕니다.

① 초기: 일단 만든다 (완벽하지 않아도 OK)
② 사용: 매주 돌려 본다
③ 개선: "코멘트를 더 짧게", "경쟁사 열도 넣어 줘" 같은 피드백을 반영
   └→ 다시 ②로. 반영할 때마다 '변경 이력'에 한 줄 남긴다.

핵심은 변경 이력입니다. 언제, 무엇을, 왜 바꿨는지 한 줄씩 남기면, 나중에 "지난달엔 왜 이렇게 했더라?"를 되짚을 수 있고, 다른 팀원에게 넘길 때도 설명이 됩니다. 광고팀의 작업 로그·수정 히스토리와 같은 개념입니다.

2-5. 운영·유지보수 — "감사와 드리프트 점검"

하네스를 오래 쓰다 보면 어긋남이 생깁니다. 이것을 **드리프트(drift, 슬금슬금 어긋남)**라고 부릅니다.

  • 팀원 정의는 "경쟁사 3곳"인데, 실제 리포트엔 5곳이 들어가고 있다.
  • 문서 템플릿을 바꿨는데, 옛 버전으로 계속 나온다.

가끔 **감사(점검)**를 해서, 하네스가 처음 설계대로 잘 돌고 있는지 확인하세요. "이 하네스 구성 점검해 줘"라고 요청하면 팀장이 현재 팀 구성과 매뉴얼을 훑어 어긋난 곳을 알려 줍니다. 분기에 한 번, 혹은 결과가 이상해질 때가 점검 타이밍입니다.

AI는 매번 조금씩 다르게 답합니다. 같은 데이터를 넣어도 코멘트 문장이나 표현이 지난주와 미묘하게 다를 수 있습니다. 이건 고장이 아니라 원래 그런 성질(비결정성)입니다. 그래서 더더욱 결과를 그냥 믿지 말고 한 번은 눈으로 확인해야 합니다. 특히 숫자와 액션 제안은 꼭 검증하세요.

3. 사용할 하네스

이번에 쓰는 팀장은 **automation-harness**입니다. 앞선 과정의 하네스들이 정해진 팀을 순서대로 돌렸다면, 이 팀장은 상황을 보고 필요한 팀원만 골라 일을 나눠 맡기고, 마지막에 문서 프로듀서에게 마감을 시킵니다.

팀 구성표

팀원(역할) 광고팀 비유 이번 업무에서 하는 일 상황별 호출
팀장(automation-harness) 총괄 팀장 업무를 조각내 배분, 진행 관리, 마감 지시 항상
데이터 담당(리서처) 데이터·조사 매체 수치 정리, 지난주 대비 비교 리포트·모니터링류
전략 담당(인사이트 에디터) 전략 기획 "왜 이런 결과인가" 인사이트 도출 해석이 필요할 때
기획 담당(인사이트 에디터/AE) 기획 다음 주 액션 제안 실행안이 필요할 때
문서 프로듀서(doc-producer) 산출물 정리 덱·문서·시트·PDF 자동 생성 항상(마감 담당)
QA 리뷰어(선택) 품질 검수 숫자·톤 최종 점검 배포 전 검증

핵심은 **"상황별 풀 + 문서 프로듀서"**입니다. 어떤 업무든 앞쪽 담당들이 내용을 만들고, 마지막에 문서 프로듀서가 통일된 포맷으로 마감합니다.

"1 요청 → 보고서 패키지"

이 하네스의 진짜 매력은 한 번의 요청으로 네 가지 문서가 한꺼번에 나온다는 점입니다.

당신의 한마디: "이번 주 리포트 만들어 줘"
                    ↓
  [데이터 요약] + [인사이트] + [액션 제안]
                    ↓  (문서 프로듀서가 받아서)
  📊 발표용 PPT 덱  📄 상세 Word 보고서  📈 성과 Excel 시트  📕 통합 PDF

트리거 브리프(지시 문장)

아래 문장을 그대로 붙여넣으면 하네스가 팀을 꾸립니다. [ ] 안만 여러분 업무에 맞게 바꾸세요.

내 팀의 [주간 캠페인 성과 리포트] 업무를 자동화하는 하네스를 구성해줘.
[데이터 요약], [인사이트 도출], [다음 주 액션 제안]을 각자 나눠 맡기고,
최종 결과를 발표용 PPT 덱, 상세 Word 보고서, 성과 지표 Excel 시트,
통합 PDF로 자동 생성해줘. 매주 같은 방식으로 재실행할 수 있게 만들어줘.

이 브리프에는 좋은 요청문의 3요소가 다 들어 있습니다 — 도메인(주간 캠페인 성과 리포트), 역할(데이터·인사이트·액션 담당), 산출물(덱·문서·시트·PDF). 이 세 가지만 챙기면 어떤 업무든 하네스로 만들 수 있습니다.

3-B. 이 하네스를 직접 만들기 — /harness 구성

이 과정의 자동화 하네스도, harness 플러그인에게 아래 한 문장을 주면 상황별 전문가를 골라 쓰고 결과를 문서로 뽑는 팀으로 자동 생성됩니다. 팀장(오케스트레이터)·팀원(에이전트)·업무 매뉴얼(스킬)이 한 번에 만들어집니다. 여러분도 아래 방법으로 직접 만들 수 있습니다.

harness 플러그인은 C1(하네스 첫걸음) 에서 이미 설치했다고 가정합니다. 아직이라면 C1의 설치 안내를 먼저 따르세요(보통 관리자가 미리 설치해 둡니다).

이 하네스를 만드는 구성 프롬프트 — 그대로 입력하세요

하네스 구성해줘. 우리 팀의 반복 업무(예: 주간 캠페인 성과 리포트)를 자동화하는 하네스가 필요해. 큰 업무를 데이터 요약·인사이트 도출·액션 제안으로 나눠 맡기고(계층적 위임), 상황에 맞는 전문가만 골라 쓰고(전문가 풀), 문서 프로듀서가 결과를 PPT·Word·Excel·PDF로 자동 생성하는 구조야. 매주 다시 실행할 수 있게 만들어줘.

업무 매뉴얼(스킬) insight-synthesis, doc-automation 등
팀장(오케스트레이터) automation-harness — 계층적 위임 + 전문가 풀

harness는 6단계(도메인 분석 → 팀 설계 → 에이전트 생성 → 스킬 생성 → 통합 → 검증)로 팀을 만들고, CLAUDE.md에 트리거 포인터를 등록합니다. 한 번 만든 뒤에도 "리서처를 하나 더 추가해줘"처럼 피드백을 주면 계속 진화합니다. 실제로 이렇게 생성된 결과물이 이 저장소의 .claude/agents/·.claude/skills/에 들어 있습니다.

만든 다음에는 아래 4장 실습의 트리거 프롬프트로 이 팀을 실행하면 됩니다.

4. 실습 — 단계별

이번 실습의 목표는 여러분 각자가 자기 팀의 반복 업무 하나를 골라 맞춤 하네스로 설계하는 것입니다. 대표 예시로 대행사의 "주간 캠페인 성과 리포트"를 함께 만들어 봅니다. 자기 업무(경쟁사 모니터링, 신규 RFP 대응, 월간 결산 등)로 바꿔도 됩니다.

4-0. 먼저: 업무 분해 캔버스 채우기

하네스를 만들기 전에, 종이 한 장에 아래 칸을 채워 보세요. 이게 설계도입니다.

칸 질문 주간 리포트 예시
① 업무 이름 무슨 반복 업무? 주간 캠페인 성과 리포트
② 입력 매번 넣는 재료는? 매체별 성과 데이터(노출·클릭·전환·비용)
③ 역할 분해 누가 무슨 조각을? 데이터 요약 / 인사이트 / 액션 제안
④ 패턴 나눠 맡기나? 골라 부르나? 계층적 위임 + 전문가 풀
⑤ 산출물 어떤 문서로? PPT + Word + Excel + PDF
⑥ 재실행 주기 얼마나 자주? 매주 월요일

4-1. 단계별 실습

각 단계에서 프롬프트(지시 문장)를 복붙하고, "기대 결과"와 실제 결과가 맞는지 확인하세요.

1단계 — 맞춤 하네스 구성

내 팀의 [주간 캠페인 성과 리포트] 업무를 자동화하는 하네스를 구성해줘.
데이터 요약, 인사이트 도출, 다음 주 액션 제안을 각자 나눠 맡기고,
최종 결과를 발표용 PPT 덱, 상세 Word 보고서, 성과 지표 Excel 시트,
통합 PDF로 자동 생성해줘. 매주 재실행할 수 있게 만들어줘.

2단계 — 이번 주 데이터 넣고 1차 실행

이번 주 데이터로 리포트를 만들어줘.
- 기간: 7월 첫째 주 (7/1~7/7)
- 매체별 성과: 메타 / 유튜브 / 검색 / 틱톡 (노출·클릭·전환·비용)
- 지난주와 비교해서 뭐가 좋아지고 나빠졌는지 표로 정리하고,
  왜 그런지 인사이트 3개와 다음 주 액션 3개를 뽑아줘.

3단계 — 문서 패키지 자동 생성

지금 결과를 4종 문서로 만들어줘.
발표용 PPT 덱(요약 중심), 상세 Word 보고서(전체 내용),
성과 Excel 시트(매체별 지표 표), 그리고 이 셋을 합친 통합 PDF.

4단계 — 피드백 반영(하네스 진화 체험)

좋아. 그런데 두 가지만 고쳐줘.
① 인사이트 코멘트를 각 2문장 이내로 짧게.
② Excel 시트에 '전주 대비 증감률(%)' 열을 추가.
이 변경을 다음 주부터 기본으로 적용하고, 변경 이력에 남겨줘.

5단계 — 재사용/배포 형태로 정리

이 하네스를 매주 그대로 쓸 수 있게 정리해줘.
매주 넣을 데이터 양식(빈 템플릿)과, 실행 방법 한 줄 안내,
지금까지의 변경 이력을 함께 남겨줘.

단계별 기대 결과

단계 하는 일 기대 결과
1 하네스 구성 데이터·인사이트·액션 담당 + 문서 프로듀서 팀이 꾸려짐
2 1차 실행 비교표 + 인사이트 3 + 액션 3 초안
3 문서화 PPT·Word·Excel·PDF 4개 파일 생성
4 진화 코멘트 짧아지고 증감률 열 추가, 변경 이력 1줄 기록
5 배포화 빈 데이터 템플릿 + 실행 안내 + 변경 이력 정리

문서 스킬 연계 (참고)

문서 프로듀서는 산출물마다 아래 업무 매뉴얼을 씁니다. 이름을 외울 필요는 없지만, "무엇으로 만들어지는지" 알아 두면 좋습니다.

산출물 사용 매뉴얼(스킬) 형식
발표용 덱 pptx .pptx
상세 보고서 docx .docx
성과 시트 xlsx .xlsx
통합 인쇄본 pdf .pdf

검증 한 번은 필수. 4종 문서가 나오면 반드시 눈으로 확인하세요. 특히 ① Excel 합계가 맞는지, ② 액션 제안이 실제로 실행 가능한지, ③ 지난주 숫자를 잘못 가져오지 않았는지. AI는 매번 조금씩 다르게 답하니, 마감 전 사람의 눈이 마지막 관문입니다.

5. 완성형 사례

대행사 "주간 캠페인 성과 리포트"를 실제로 자동화한 완성형 사례를 별도 문서로 정리했습니다. 업무 분해, 패턴 선택 근거, 자동 생성되는 4종 문서 구성, 매주 재실행 방법, 그리고 실제처럼 집필한 샘플 주간 리포트 전문이 담겨 있습니다.

👉 C2 완성형 사례 — 주간 캠페인 성과 리포트 자동화

핵심 발췌 — 샘플 리포트 요약 (예시 데이터)

제로톡 여름 캠페인 · 7월 1주차 주간 리포트 (※ 모든 수치는 예시 데이터)

이번 주 총 전환 1,240건(전주 대비 +18%), 전환당 비용 8,900원(-12%). 유튜브 조회 기반 캠페인이 전환 효율을 끌어올린 반면, 틱톡은 노출은 늘었으나 전환이 정체. 검색은 브랜드 키워드 유입이 안정적으로 유지됨.

다음 주 액션 3개: ① 유튜브 예산 15% 증액, ② 틱톡 소재 3종 교체 A/B, ③ 검색 제외 키워드 정비로 낭비 클릭 축소.

전체 지표표·인사이트·액션 상세는 사례 문서에서 확인하세요.

6. 자주 하는 실수

실수 이렇게 하세요
너무 큰 업무를 한 번에 자동화하려 함 (예: "우리 팀 전체 업무를 자동화해 줘") 반복 업무 하나로 좁히세요. "주간 리포트" 하나부터. 잘 돌면 그때 다음 업무로 넓힙니다.
문서 포맷이 매번 달라짐 (덱 구성이 주마다 제각각) 팀 표준 템플릿을 한 번 정해 고정하세요. "지난주와 같은 틀로"라고 요청하면 됩니다.
변경 이력을 안 남김 고칠 때마다 "변경 이력에 남겨 줘"를 붙이세요. 나중에 되짚고, 남에게 넘길 때 설명이 됩니다.
결과(특히 숫자)를 검증 없이 그대로 사용 마감 전 한 번은 의심하세요. Excel 합계, 지난주 비교값, 액션의 실행 가능성을 눈으로 확인.
