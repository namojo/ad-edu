# C2 · 리서치·인사이트 하네스

> 하루 걸리던 카테고리 조사를, 네 명이 동시에 파고들어 한 시간 만에 인사이트로.

## 0. 이 과정 한눈에

- **대상**: 기획/AE, 전략, 마케터 (조사·인사이트를 다루는 실무자)
- **소요 시간**: 약 150분
- **선수 과정**: C1 하네스 첫걸음 (하네스 개념·첫 실행 경험 필요)
- **사용 하네스**: `research-harness` — 리서처 4명이 동시에 조사하고, 인사이트 에디터 1명이 하나로 통합
- **얻어가는 것(산출물)**:
  - "제로톡" 카테고리 진단 **인사이트 보고서 1부** (시장·소비자·경쟁·트렌드 + 인사이트 3개)
  - 다시 쓸 수 있는 **나만의 리서치 하네스**(팀)
  - 상충 정보를 다루는 **교차검증 습관**

---

## 1. 왜 이 하네스인가

**실무 문제.** 새 캠페인을 앞두고 카테고리를 진단하라는 지시가 떨어졌습니다.
시장 규모도 봐야 하고, 소비자가 뭘 불편해하는지도 알아야 하고, 경쟁사가 뭐라고 떠드는지,
요즘 트렌드는 뭔지까지 살펴야 합니다. 혼자서 이걸 순서대로 하면 **하루가 꼬박** 갑니다.
게다가 급하게 하다 보면 한두 각도는 얕게 훑고 넘어가기 쉽죠.

**해결.** 이 일을 **네 명이 동시에** 하면 어떨까요? 한 명은 시장만, 한 명은 소비자만,
한 명은 경쟁사만, 한 명은 트렌드만 파고듭니다. 네 조사가 끝나면 편집 담당 한 명이
결과를 모아 겹치는 부분은 신뢰를 높이고, 엇갈리는 부분은 출처를 함께 남긴 뒤
**"다음 캠페인 방향" 인사이트 3개**로 정리합니다. 이것이 리서치 하네스입니다.

| | Before (혼자·순차) | After (하네스·병렬) |
|---|---|---|
| 방식 | 시장→소비자→경쟁→트렌드 한 명이 차례로 | 네 명이 네 각도를 동시에 |
| 시간 | 반나절~하루 | 30~60분 |
| 관점 | 한 사람 시각에 치우침 | 각 분야 전담 → 관점 다양성 |
| 상충 정보 | 대충 하나 골라 넘어감 | 출처 병기로 판단 근거 보존 |
| 결과물 | 조사 자료 뭉치 | 근거가 붙은 인사이트 3개 |

---

## 2. 개념 이해 — 팬아웃 / 팬인

이번 패턴의 이름은 **팬아웃/팬인**입니다. 선풍기(fan) 날개가 **활짝 펼쳐졌다가(팬아웃) 다시 한 점으로 모이는(팬인)**
모습을 떠올리면 쉽습니다.

- **팬아웃 (동시에 흩어짐)**: 하나의 주제를 여러 갈래로 나눠 **동시에** 조사합니다.
  광고팀으로 치면, 팀장이 회의에서 "너는 시장, 너는 소비자, 너는 경쟁, 너는 트렌드 맡아. 지금 각자 시작"이라고
  일을 쪼개 뿌리는 순간입니다. 순서를 기다리지 않으니 **빠르고**, 각자 자기 분야만 파니 **깊습니다**.
- **팬인 (하나로 모임)**: 흩어졌던 결과를 **한 명이 받아 통합**합니다.
  네 사람의 조사 노트를 모아 겹치는 이야기는 강조하고, 엇갈리는 이야기는 나란히 적어 정리 담당(인사이트 에디터)이
  **한 편의 보고서**로 엮습니다.

**왜 좋은가 — 두 마리 토끼.** 순차로 하면 속도를 얻는 대신 관점이 좁아지고, 여러 명에게 따로 시키면
관점은 넓어지는 대신 취합이 번거롭습니다. 팬아웃/팬인은 **속도(동시 진행)**와 **관점 다양성(분야 전담)**을
한꺼번에 잡습니다.

**교차검증 — 신뢰를 만드는 규칙.** 통합할 때 핵심은 "누가 맞나 하나만 고르기"가 아닙니다.
- 여러 조사가 **같은 결론**을 가리키면 → 그 주장의 **신뢰도를 높입니다.**
- 조사끼리 **엇갈리면** → 지우지 말고 **출처를 함께 적어 둡니다.** (예: "A 자료는 15%, B 자료는 12%")
  판단은 사람이 하되, 그 판단의 근거를 남겨 두는 것이죠.

**근거 기반 인사이트 — 주장 → 근거 → 시사점.** 좋은 인사이트는 "느낌"이 아니라 **구조**가 있습니다.
- **주장**: 무엇을 하자는 한 줄 (예: "건강 말고 '맛의 해방'으로 가자")
- **근거**: 어떤 조사가 그렇게 말하는가 (경쟁·소비자·트렌드 데이터)
- **시사점**: 그래서 캠페인에서 뭘 하나 (메시지·타깃·콘텐츠로 연결)

> AI는 같은 질문에도 매번 조금씩 다르게 답합니다(비결정성). 그러니 조사 결과를 그대로 믿지 말고,
> **한 번은 원출처로 되짚어 확인**하는 습관을 이 과정 내내 함께 익힙니다.

## 3. 사용할 하네스 — `research-harness`

이번에 부르는 하네스는 **리서치 하네스**입니다. 팀장(오케스트레이터)이 리서처 4명을 **동시에** 풀어
조사시키고, 인사이트 에디터 1명이 결과를 모읍니다.

### 팀 구성표

| 역할(팀원) | 이 사람이 하는 일 | 광고팀 비유 |
|---|---|---|
| **팀장** (research-harness) | 조사를 네 갈래로 쪼개 뿌리고, 결과를 모으는 지휘 | 리서치 총괄 |
| 시장 리서처 | 시장 규모·성장성·세그먼트 조사 | 시장 조사 담당 |
| 소비자 리서처 | 소비자 인식·구매 동기·페인포인트 조사 | 소비자 인사이트 담당 |
| 경쟁 리서처 | 경쟁사 포지셔닝·메시지·타깃 조사 | 경쟁 분석 담당 |
| 트렌드 리서처 | 카테고리 최신 트렌드·시그널 조사 | 트렌드 워처 |
| **인사이트 에디터** | 네 결과 교차검증 → 인사이트 3개로 통합 | 전략 기획 |

*앞의 네 명은 **동시에(팬아웃)** 일하고, 마지막 인사이트 에디터가 **하나로 모읍니다(팬인)**.*

### 트리거 브리프 (이 문장을 그대로 입력)

```text
리서치 하네스를 구성해줘. 하나의 브랜드/카테고리를 여러 각도에서 동시에 조사하는 팀이 필요해 — 시장 규모와 성장성, 소비자 인식과 페인포인트, 경쟁사 포지셔닝, 최신 트렌드. 각자 조사한 뒤 교차검증하고, 상충하는 정보는 출처를 함께 남겨서 '캠페인 방향 인사이트 3개 + 근거'가 담긴 종합 보고서를 만들어줘.
```

이 브리프에는 좋은 요청문의 **3요소**가 들어 있습니다.
- **도메인**: 브랜드/카테고리를 여러 각도에서 조사
- **역할**: 시장·소비자·경쟁·트렌드 리서처 + 통합 에디터
- **산출물**: 캠페인 방향 인사이트 3개 + 근거가 담긴 종합 보고서

### 더 깊게 파고 싶을 때 — deep-research 연계

특정 주장 하나(예: "시장이 정말 연 15% 성장하나?")를 더 파야 할 때는 **deep-research**를 이어서 쓸 수 있습니다.
여러 소스를 훑어 교차검증하는 심화 조사 도구로, 리서치 하네스의 결론 중 **의심스러운 한 가지를 다시 검증**하는 데
적합합니다. (선택 단계 — 4장 5단계에서 안내)

이 과정에서 쓰는 리서치 하네스도, harness 플러그인에게 아래 한 문장을 주면 리서처 4명과 인사이트 에디터로 이루어진 팀이 통째로 만들어집니다. 팀장(오케스트레이터)·팀원(에이전트)·업무 매뉴얼(스킬)이 한 번에 만들어집니다. 여러분도 아래 방법으로 직접 만들 수 있습니다.

```text
/plugin marketplace add revfactory/harness
/plugin install harness-marketplace
```

> 사내 마켓플레이스를 쓰면 그 경로로 바꾸세요.

**② 이 하네스를 만드는 구성 프롬프트 — 그대로 입력하세요**

```text
하네스 구성해줘. 하나의 카테고리를 여러 각도에서 동시에 조사하는 리서치 팀이 필요해. 시장·소비자·경쟁·트렌드를 각각 맡는 리서처 네 명이 병렬로 조사(팬아웃)하고, 인사이트 에디터 한 명이 교차검증해 하나의 보고서로 종합(팬인)하는 구조야. 상충하는 정보는 출처를 병기하고, '주장→근거→시사점' 인사이트를 결과로 줘. 매번 다른 브랜드로 다시 실행할 수 있게.
```

**③ 무엇이 만들어지나요**

| 종류 | 생성물 |
|---|---|
| 팀원(에이전트) | `researcher`(시장·소비자·경쟁·트렌드 4역할), `insight-synthesizer`(인사이트 에디터) |
| 업무 매뉴얼(스킬) | `market-research`, `insight-synthesis` |
| 팀장(오케스트레이터) | `research-harness` — 팬아웃/팬인 |

> harness는 **6단계**(도메인 분석 → 팀 설계 → 에이전트 생성 → 스킬 생성 → 통합 → 검증)로 팀을 만들고, `CLAUDE.md`에 트리거 포인터를 등록합니다. 한 번 만든 뒤에도 "리서처를 하나 더 추가해줘"처럼 피드백을 주면 **계속 진화**합니다. 실제로 이렇게 생성된 결과물이 이 저장소의 `.claude/agents/`·`.claude/skills/`에 들어 있습니다.

**만든 다음에는** 아래 4장 실습의 트리거 프롬프트로 이 팀을 실행하면 됩니다.

## 4. 실습 — "제로톡" 카테고리 진단

가상 브랜드 **"제로톡"**(제로 슈거 스파클링 음료)의 카테고리를 진단합니다.
아래 프롬프트를 순서대로 복붙하며 따라오세요.

**1단계 — 하네스 구성**

```text
리서치 하네스를 구성해줘. 하나의 브랜드/카테고리를 여러 각도에서 동시에 조사하는 팀이 필요해 — 시장 규모와 성장성, 소비자 인식과 페인포인트, 경쟁사 포지셔닝, 최신 트렌드. 각자 조사한 뒤 교차검증하고, 상충하는 정보는 출처를 함께 남겨서 '캠페인 방향 인사이트 3개 + 근거'가 담긴 종합 보고서를 만들어줘.
```

**2단계 — 조사 대상 입력**

```text
대상은 '제로톡'이라는 제로 슈거 스파클링 음료 브랜드가 속한 '제로 슈거 음료' 카테고리야. 국내 시장, 최근 3년 기준으로 조사해줘. (실제 데이터가 없으면 예시 데이터임을 명확히 표시해서 형식만 채워줘.)
```

**3단계 — 인사이트를 구조에 맞춰 정리 요청**

```text
캠페인 방향 인사이트 3개를 각각 '주장 → 근거 → 시사점' 형식으로 정리해줘. 근거에는 어느 조사(시장/소비자/경쟁/트렌드)에서 나왔는지 표시해줘.
```

**4단계 — 상충 정보 확인**

```text
네 조사 결과 중 서로 엇갈리는 정보가 있으면 하나로 합치지 말고, 각각 어느 출처에서 나왔는지 나란히 표시해줘.
```

**5단계 (선택) — 심화 재검증**

```text
deep-research로 '시장 성장률'과 '핵심 타깃' 주장만 다시 한 번 교차검증해줘. 근거가 약하면 약하다고 알려줘.
```

### 각 단계 기대결과

| 단계 | 입력 | 이렇게 나오면 정상 |
|---|---|---|
| 1 | 하네스 구성 브리프 | 리서처 4명 + 인사이트 에디터로 팀이 짜임 |
| 2 | 조사 대상 입력 | 네 각도 조사가 **동시에(팬아웃)** 진행 |
| 3 | 인사이트 정리 요청 | 인사이트 3개가 주장→근거→시사점 구조로 통합(팬인) |
| 4 | 상충 정보 확인 | 엇갈리는 수치가 **출처 병기**로 남음(예: 15% vs 12%) |
| 5 | 심화 재검증(선택) | 두 주장의 근거 강도를 다시 평가 |

> AI는 실행할 때마다 표현·수치가 조금씩 달라집니다. 팀 구성과 보고서 뼈대가 위와 같으면 정상입니다.
> 그리고 나온 결과는 **한 번은 의심하고** 원출처를 확인하세요(4단계·5단계가 그 습관을 만들어 줍니다).

(모든 수치·출처는 교육용 **예시 데이터**입니다.)

**핵심 발췌 — 캠페인 방향 인사이트 3개**

1. **"제로 슈거"를 반복하지 말고 "맛의 죄책감 해방"으로 전선을 옮겨라**
   — 경쟁 메시지가 "제로=건강"으로 다 똑같아졌고(경쟁), 소비자 동기 2위가 "죄책감 없이 즐김"(소비자),
   "guilt-free" 감정 프레임이 확산(트렌드). → 메시지 축을 건강에서 **즐거움·해방**으로.
2. **2030 여성이 아니라 "3040 건강 관심 남성"의 미개척 수요를 노려라**
   — 경쟁 3사가 2030 여성에 과밀(경쟁), 남성 헬스 문화 확산(트렌드), **상충 데이터**에서 남성 구매액 성장률 1위.
   → 남성 겨냥 라인 신설(단, 상충 근거라 소규모 A/B로 먼저 검증).
3. **최대 페인포인트 "성분 불신"을 투명성으로 정면 돌파하라**
   — 재구매 이탈 1위가 인공감미료 불신 47%(소비자), 클린 라벨 트렌드(트렌드), 경쟁사 누구도 안 함(경쟁).
   → 성분 투명성 콘텐츠를 신뢰 축으로.

세 인사이트 모두 **주장 → 근거(어느 조사) → 시사점**의 구조를 갖췄고, 상충했던 "핵심 타깃"은
지우지 않고 **출처를 병기**해 인사이트 2의 근거로 살렸다는 점에 주목하세요.

## 6. 자주 하는 실수

- **조사 범위가 너무 넓어 얕아진다.**
  → 이렇게 하세요: **카테고리·기간·지역을 먼저 못 박으세요.** ("국내 / 최근 3년 / 제로 슈거 음료"처럼).
  범위가 좁아야 각 리서처가 깊이 팝니다.
- **상충하는 정보를 임의로 하나만 채택한다.**
  → 이렇게 하세요: 엇갈리면 **둘 다 남기고 출처를 병기**하세요. "누가 맞나"가 아니라 "어떤 근거로 그렇게 보나"를
  남기는 것이 리서치의 신뢰입니다.
- **근거 없는 '느낌' 인사이트를 쓴다.**
  → 이렇게 하세요: 모든 인사이트에 **"어느 조사에서 나왔나"를 붙이세요.** 근거를 못 대면 인사이트가 아니라 추측입니다.
- **AI 결과를 그대로 발표 자료에 붙인다.**
  → 이렇게 하세요: **발표 전 한 번은 원출처를 직접 확인**하세요. AI는 매번 조금씩 다르게 답하고, 가끔 틀립니다.

## 7. 체크리스트 & 자기평가

**완주 체크리스트**
- [ ] 리서치 하네스가 리서처 4명 + 인사이트 에디터로 구성되었다
- [ ] 시장·소비자·경쟁·트렌드가 **동시에(팬아웃)** 조사되는 것을 관찰했다
- [ ] 인사이트 에디터가 결과를 **하나로(팬인)** 통합했다
- [ ] 상충 정보를 **출처 병기**로 처리했다(임의 채택하지 않았다)
- [ ] 인사이트 3개가 **주장 → 근거 → 시사점** 구조를 갖췄다
- [ ] 최소 한 가지 주장을 **원출처로 되짚어 확인**했다

**자기평가 루브릭 요약 (R-C2 · 통과 70%)**

| 평가 항목 | 비중 | 우수 기준 |
|---|---|---|
| 패턴 적용(팬아웃/팬인) | 25% | 병렬 조사 → 통합을 의도대로 구성 |
| 교차검증·출처 | 30% | 상충 정보 출처 병기, 인사이트마다 근거 링크 |
| 인사이트 품질 | 30% | 주장→근거→시사점 구조, 실행 가능성 |
| 산출물 완성도 | 15% | 보고서 가독성·문서화 |
