# C5 · 미디어·캠페인 플래닝 하네스

> 한 명의 감독이 캠페인 전체를 지휘합니다. 타겟·채널·예산·A/B를 나눠 맡기고, 다시 하나의 미디어 플랜으로 취합합니다.

## 0. 이 과정 한눈에

- 대상: 미디어 플래너, 퍼포먼스 마케터, AE
- 소요: 약 150분
- 선수과정: C1(하네스 첫걸음)
- 사용 하네스: `media-harness` (감독자 패턴)
- 얻어가는 것(산출물): 채널별 예산 배분표(Excel), 캠페인 브리프, A/B 테스트 계획, 재사용 가능한 미디어 하네스

---

## 1. 왜 이 하네스인가

미디어 플랜 하나를 만들려면 생각보다 많은 일이 동시에 굴러갑니다. 타겟을 정의하고, 채널을 고르고, 예산을 나누고, 성과 측정과 A/B까지 설계해야 합니다. 이 일들은 서로 얽혀 있어서, 하나만 바꿔도 나머지가 흔들립니다. 예산을 유튜브 쪽으로 더 옮기면 도달 예측이 바뀌고, 그러면 KPI 목표도 다시 손봐야 합니다.

**Before** — 담당자 한 명이 엑셀을 열어 채널별로 예산을 손으로 나눕니다. 합계가 안 맞아 다시 계산하고, KPI는 나중에 끼워 넣고, A/B 계획은 시간이 없어 생략됩니다. 반나절이 그냥 갑니다.

**After** — "감독자"에게 큰 목표(총예산·기간·타겟)만 넘기면, 감독자가 타겟 담당·채널 담당·예산 담당에게 일을 나눠 맡기고, 결과를 하나의 미디어 플랜으로 취합해 줍니다. 사람은 큰 방향을 정하고, 나온 숫자를 검산하는 데 집중합니다.

---

## 2. 개념 이해

### 감독자 패턴 = 캠페인 매니저 한 명이 지휘

감독자 패턴은 광고 대행사의 **캠페인 매니저(감독)** 를 떠올리면 쉽습니다. 감독은 직접 카피를 쓰거나 매체를 사지 않습니다. 대신 "너는 타겟 정의, 너는 채널 믹스, 너는 예산 배분"처럼 일을 **동적으로 나눠 맡기고**, 진행 상태를 관리하다가, 각자의 결과를 모아 하나의 플랜으로 합칩니다.

앞 과정에서 배운 파이프라인(앞사람 결과를 뒷사람이 이어받는 순차 협업)과 다릅니다. 감독자 패턴은 중앙의 감독이 **상황을 보고 그때그때 누구에게 무엇을 맡길지 결정**한다는 점이 핵심입니다. 예산 논의 중에 "채널을 다시 봐야겠다" 싶으면 감독이 채널 담당을 다시 부릅니다.

### 미디어 태스크 분해 — 네 덩어리로 나눈다

감독자는 미디어 플래닝을 네 덩어리로 쪼갭니다.

| 덩어리 | 무엇을 정하나 |
|---|---|
| 타겟 | 누구에게 말할 것인가 (1차/2차 타겟, 페르소나) |
| 채널 | 어디서 말할 것인가 (채널 믹스) |
| 예산 | 각 채널에 얼마를 쓸 것인가 (배분표) |
| 측정 | 성과를 무엇으로 볼 것인가 (KPI·A/B) |

### 채널믹스 로직

채널은 "역할"로 고릅니다. 도달을 넓히는 채널(유튜브·인스타), 전환을 수확하는 채널(네이버 검색), 바이럴을 노리는 채널(틱톡), 지역 인지를 얹는 채널(옥외)을 목표 비중에 맞게 섞습니다. 타겟이 20대라면 그들이 실제로 머무는 채널에 예산을 더 싣는 것이 기본입니다.

### A/B 설계 — 가설·변인·성공지표

A/B 테스트는 "무엇이 더 나은지 데이터로 확인하는" 실험입니다. 세 가지를 반드시 정합니다.

- 가설: "감성 소구가 기능 소구보다 완주율이 높을 것이다" 처럼 **미리 예측**을 적습니다.
- 변인: **한 번에 하나만** 바꿉니다. 훅 메시지만 바꾸거나, 커버 이미지만 바꿉니다. 여러 개를 동시에 바꾸면 무엇 때문에 결과가 달라졌는지 알 수 없습니다.
- 성공지표: 완주율·CTR·전환율처럼 **숫자로 판정 가능한 기준**을 정합니다.

### 예산 합계 검산 — 습관으로 만든다

미디어 플랜에서 가장 흔한 실수가 **합계 불일치**입니다. 채널별 금액을 다 더했을 때 총예산과 정확히 같아야 하고, 비중(%)의 합은 100%가 되어야 합니다. AI가 만든 표라도 이 검산은 사람이 직접 합니다.

> AI는 매번 조금씩 다르게 답합니다(비결정성). 같은 브리프를 넣어도 배분이 달라질 수 있어요. 그래서 나온 숫자는 그대로 믿지 말고, 합계와 KPI 논리를 **최소 한 번은 사람이 검증**합니다.

## 3. 사용할 하네스

이 과정은 `media-harness` 오케스트레이터를 씁니다. 감독자 패턴으로 아래 팀을 지휘합니다.

### 팀 구성표

| 팀원(역할) | 광고팀 비유 | 하는 일 |
|---|---|---|
| 미디어 플래너 (감독) | 캠페인 매니저 | 타겟·채널·예산·측정 태스크를 나눠 맡기고 취합. 전체 지휘 |
| 리서처 | 조사 담당 | 채널별 예상 단가·소비자 매체 이용 행태 등 근거 조사 |
| 문서 프로듀서 | 산출물 정리 담당 | 최종 배분표를 Excel 미디어 플랜 시트로 정리(문서 자동화) |

- 감독(미디어 플래너)이 중앙에서 진행을 관리하고, 리서처는 근거를, 문서 프로듀서는 산출물을 담당합니다.
- 감독에게는 세세한 지시 대신 **큰 목표(총예산·기간·타겟)만** 주고 위임하는 것이 요령입니다.

### 오케스트레이터 트리거 문장

```text
미디어·캠페인 플래닝 하네스를 구성해줘. 중앙에서 캠페인을 지휘하는 감독자 에이전트가 타겟 정의, 채널 믹스, 예산 배분, A/B 테스트 계획을 각 전문가에게 나눠 맡기고 취합해줘. 총 예산과 목표를 주면 '채널별 예산 배분표(엑셀), 타겟 정의, A/B 테스트 계획'이 담긴 미디어 플랜을 만들어줘.
```

## 3-B. 이 하네스를 직접 만들기 — `/harness` 구성

이 과정의 미디어 하네스도, harness 플러그인에게 아래 한 문장을 주면 중앙에서 지휘하는 감독자 구조의 팀으로 자동 생성됩니다. 팀장(오케스트레이터)·팀원(에이전트)·업무 매뉴얼(스킬)이 한 번에 만들어집니다. 여러분도 아래 방법으로 직접 만들 수 있습니다.

> harness 플러그인은 **C1(하네스 첫걸음)** 에서 이미 설치했다고 가정합니다. 아직이라면 C1의 설치 안내를 먼저 따르세요(보통 관리자가 미리 설치해 둡니다).

**이 하네스를 만드는 구성 프롬프트 — 그대로 입력하세요**

```text
하네스 구성해줘. 캠페인 미디어 플랜을 짜는 팀이 필요해. 중앙에서 지휘하는 미디어 플래너(감독자)가 타겟·채널·예산·A/B를 나눠 맡기고, 리서처가 근거를 대고, 문서 프로듀서가 결과를 엑셀 미디어 플랜으로 정리하는 구조야. 총예산을 주면 채널별 배분표와 A/B 계획을 결과로 줘.
```

**무엇이 만들어지나요**

| 종류 | 생성물 |
|---|---|
| 팀원(에이전트) | `media-planner`(미디어 플래너·감독), `researcher`(리서처), `doc-producer`(문서 프로듀서) |
| 업무 매뉴얼(스킬) | `media-planning`, `market-research`, `doc-automation` |
| 팀장(오케스트레이터) | `media-harness` — 감독자(동적 배분) |

> harness는 **6단계**(도메인 분석 → 팀 설계 → 에이전트 생성 → 스킬 생성 → 통합 → 검증)로 팀을 만들고, `CLAUDE.md`에 트리거 포인터를 등록합니다. 한 번 만든 뒤에도 "리서처를 하나 더 추가해줘"처럼 피드백을 주면 **계속 진화**합니다. 실제로 이렇게 생성된 결과물이 이 저장소의 `.claude/agents/`·`.claude/skills/`에 들어 있습니다.

**만든 다음에는** 아래 4장 실습의 트리거 프롬프트로 이 팀을 실행하면 됩니다.

## 4. 실습 — 단계별

시나리오: **신제품 음료 "제로톡" 런칭 캠페인, 총예산 5,000만원**의 미디어 플랜을 만듭니다.

아래 브리프를 그대로 복붙해 감독자에게 넘기세요. (도메인·역할·산출물 3요소가 모두 담겨 있습니다.)

```text
제로톡(제로 슈거 스파클링 음료) 런칭 캠페인의 미디어 플랜을 만들어줘.
- 총예산: 5,000만원 / 기간: 8주 / 목표: 런칭 인지도 + 초기 구매 전환
- 1차 타겟: 20대 수도권 건강 관심층, 2차 타겟: 30대 헬스 관심층
감독자가 타겟 정의·채널 믹스·예산 배분·A/B 계획을 나눠 맡겨서,
'채널별 예산 배분표(채널×예산×비중×예상 KPI) + 타겟 정의 + A/B 테스트 2~3개'를 만들어줘.
채널 금액 합계는 반드시 5,000만원과 정확히 일치시켜줘.
```

| 단계 | 할 일 | 기대 결과 |
|---|---|---|
| 1 | 미디어 하네스 구성 | 감독(미디어 플래너) + 리서처 + 문서 프로듀서 팀이 만들어짐 |
| 2 | 위 브리프 입력 (예산·기간·타겟) | 감독이 태스크를 네 덩어리로 분배 |
| 3 | 감독의 태스크 분배·진행 관리 관찰 | "누가 무엇을 맡는지"가 눈에 보임 |
| 4 | 채널 믹스·예산 배분표 수령 | 채널×예산×비중×KPI 표, 합계 5,000만원 |
| 5 | 합계 검산 | 채널 금액 합 = 5,000만원, 비중 합 = 100% 확인 |
| 6 | 보수형/공격형 시나리오 비교 재요청 | 같은 예산의 두 대안 표 |
| 7 | A/B 테스트 계획 + 캠페인 브리프 취합 | 가설·변인·성공지표가 담긴 최종 플랜 |

**Excel로 내보내기**: 배분표가 확정되면 문서 프로듀서(doc-automation 연계)에게 "이 배분표를 Excel 미디어 플랜 시트로 정리해줘"라고 요청하면, 채널×예산×KPI가 담긴 `.xlsx` 파일로 산출됩니다. 발표·공유용으로 바로 씁니다.

> 검증 1회 필수: 나온 배분표의 합계를 사람이 직접 더해 5,000만원과 일치하는지 확인하세요. AI가 만든 표라도 이 한 번의 검산은 생략하지 않습니다.

## 5. 완성형 사례

"제로톡" 런칭 미디어 플랜의 전체 산출물은 아래 파일에 있습니다.

- 사례 전문: [C5_미디어플랜.md](../사례/C5_미디어플랜.md)

핵심 배분표만 발췌하면 다음과 같습니다(추천안·균형형, **예시 데이터**).

| 채널 | 예산(원) | 비중 | 예상 핵심 KPI |
|---|---:|---:|---|
| 유튜브 (인스트림+쇼츠) | 15,000,000 | 30% | 조회 250만 · CPV 6원 |
| 인스타그램 (릴스+피드) | 12,500,000 | 25% | 도달 320만 · 저장 1.2만 |
| 네이버 (검색+GFA) | 9,500,000 | 19% | 유입 9.5만 · 전환 3,800 |
| 틱톡 | 8,000,000 | 16% | 조회 400만 · 참여 8천 |
| 디지털 옥외(DOOH) | 5,000,000 | 10% | 도달 180만(추정) |
| **합계** | **50,000,000** | **100%** | — |

합계 검산: 15,000,000 + 12,500,000 + 9,500,000 + 8,000,000 + 5,000,000 = **50,000,000원 (일치)**. 사례 파일에는 보수형/공격형 비교와 A/B 테스트 3개, 캠페인 브리프가 모두 담겨 있습니다.

## 6. 자주 하는 실수

- 예산 합계 불일치 → 배분표를 만들 때마다 채널 금액 합과 비중 합(100%)을 **직접 더해 검산**하세요.
- KPI 없는 채널 배분 → 모든 채널에 목표 지표(도달·CPV·전환 등)를 하나씩 붙이세요. 지표 없는 예산은 평가할 수 없습니다.
- 감독에게 과도한 세부 지시 → 큰 목표(예산·기간·타겟)만 주고 나머지는 위임하세요. 감독의 판단 여지를 남깁니다.
- A/B에서 변인을 여러 개 동시 변경 → 한 번에 하나만 바꿔야 원인을 구분할 수 있습니다.
