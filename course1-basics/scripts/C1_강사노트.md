# C1 「하네스 첫걸음」 — 강사 노트

> 과정 1 · 기초 · 120분 · 대상: 광고 실무자(비개발자) 전원 공통
> 실행 모드: 강의 + 라이브 시연 + hands-on 실습 + 리뷰 (4비트)

---

## 0. 사전 점검 (강의 시작 15분 전)

- [ ] 수강생 PC: Claude Code 설치·로그인 확인
- [ ] harness 플러그인 설치 상태 확인 (`/plugin` 목록에 harness)
- [ ] 에이전트 팀 기능 활성화 (`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`)
- [ ] 실습 폴더(`~/harness-lab/`) 생성
- [ ] 교보재 배포: 강의덱(PPT), 핸드아웃(PDF), 워크시트(Excel), 프롬프트 모음
- [ ] **폴백 준비**: 네트워크/모델 지연 대비 — 사전 생성한 "제로톡" 결과(시장요약+헤드라인) 캡처를 열어둔다
- [ ] 강사 화면: 하네스 실행을 크게 보여줄 수 있게 폰트 확대

## 2. 라이브 시연 스크립트 (45–55분 구간에서 강사가 먼저)

강사가 실습 프롬프트를 **직접 한 번 시연**한 뒤 수강생이 따라 하게 한다.

1. "하네스 구성해줘" 프롬프트 입력 → 하네스가 **도메인을 분석**하는 것을 소리 내어 해설
2. 리서처 + 카피라이터 2인 팀이 제안되면 → "보세요, 팀장이 팀원을 뽑았죠?"
3. `.claude/agents/` 와 `.claude/skills/` 를 열어 → **"누가(에이전트)"와 "어떻게(스킬)"가 파일로 분리**됨을 보여준다
4. 결과(시장요약 + 헤드라인 5개)를 함께 읽고 → "이게 파이프라인(조사→카피)이에요"
5. "헤드라인을 더 위트있게" 재요청 → **결과가 바뀌는 순간**을 강조 ("쓸수록 진화")

## 3. 강의 포인트 (용어를 만나면 즉시 광고 비유로 치환)

| 용어 | 이렇게 말하세요 |
|------|----------------|
| 에이전트 | "팀원" |
| 오케스트레이터 | "팀장" |
| 스킬 | "업무 매뉴얼" |
| 프롬프트 | "브리프 / 지시 문장" |
| 파이프라인 | "조사한 사람 결과를 카피라이터가 받는 순서" |

- **AI 비결정성 안내**: "AI는 매번 조금씩 다르게 답합니다. 옆 사람과 결과가 달라도 정상이에요."
- **검증 습관**: "결과를 그냥 믿지 마세요. 한 번은 의심하고, 마음에 안 들면 다시 요청하세요."

## 4. 자주 나오는 질문 (FAQ)

**Q. 팀이 아니라 그냥 한 번에 답이 나와요.**
→ 에이전트 팀 기능이 꺼져 있습니다. 환경변수(`CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`)를 확인하세요.

**Q. "하네스 구성해줘"가 작동하지 않아요.**
→ 정확한 표현을 사용하세요. harness 플러그인이 설치·활성화됐는지 먼저 확인.

**Q. 결과가 너무 길거나 엉뚱해요.**
→ 요청문에 **산출물 형식·분량**을 명시하세요("1페이지", "5개").

**Q. 이거 코딩 배워야 하나요?**
→ 아니요. 여러분은 **팀장 역할**만 합니다. 지시하고, 검수하고, 다시 요청하는 것이 전부입니다.

**Q. 회사 자료를 넣어도 되나요?**
→ 보안 정책을 따르세요. 실습은 공개 정보/가상 브랜드("제로톡")로 진행합니다.

## 5. 트러블슈팅 (실습 중)

| 증상 | 조치 |
|------|------|
| 실행이 멈춤/지연 | 폴백 캡처로 개념 설명 이어가고, 백그라운드로 재시도 |
| 수강생마다 결과 편차 | "정상"이라고 안내, 우수 사례를 화면 공유 |
| 폴더가 안 보임 | 파일 탐색기에서 실습 폴더 경로 함께 확인 |
| 헤드라인 품질 낮음 | 톤/타겟을 프롬프트에 추가해 재요청 시연 |

## 6. 리뷰 진행 (105–115분)

- 2~3명에게 헤드라인 5개 화면 공유 요청 → 서로 다른 결과를 비교
- "3요소(도메인·역할·산출물)를 다 넣었나요?" 자가 점검
- 워크시트 **자기평가 루브릭** 작성 안내 (70점 이상 통과)
