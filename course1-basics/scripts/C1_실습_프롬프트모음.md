# C1 실습 — 복사·붙여넣기 프롬프트 모음

> Claude Code에 그대로 붙여넣어 사용하세요. 모든 프롬프트는 **도메인·역할·산출물** 3요소를 담고 있습니다.

---

## 1. 메인 실습 — "제로톡" 런칭 팀 (필수)

```
하네스 구성해줘. 신제품 음료 런칭을 돕는 작은 팀이 필요해 — 시장/경쟁 상황을 빠르게 조사하는 리서처와, 그 조사를 바탕으로 광고 헤드라인을 뽑는 카피라이터. 조사 요약 1페이지와 헤드라인 5개를 결과로 줘.
```

**기대 결과:** 리서처 + 카피라이터 2인 에이전트 팀 구성 → 시장요약 1p + 헤드라인 5개
**패턴:** 파이프라인 (조사 → 카피)

---

## 2. 미니 개선 — 톤 바꿔 재요청 (진화 체험)

메인 실습 결과가 나온 뒤 이어서 입력:

```
헤드라인을 더 위트있게, 20대가 SNS에서 공유하고 싶어할 톤으로 5개 다시 뽑아줘.
```

**포인트:** 같은 팀이 **톤만 바꿔 재생성** → "하네스는 쓸수록 진화한다"를 체감.

---

## 3. 3요소 연습용 변형 (선택)

### (a) 산출물을 바꿔 요청
```
방금 만든 팀에게, 조사 요약을 슬로건 후보 3개 + 타겟 한 줄 정의로 바꿔서 다시 정리해달라고 해줘.
```

### (b) 역할을 추가
```
이 팀에 '브랜드 톤 검수자'를 한 명 추가해줘. 헤드라인이 브랜드 톤(젊고 위트있게)에 맞는지 검수하고, 안 맞으면 다시 뽑으라고 지시하는 역할이야.
```
> 이때 팀 협업 방식이 **생성-검증** 쪽으로 바뀌는 것을 관찰 (C3 예고).

---

## 4. 나만의 업무로 바꾸기 (실습 후 자율)

아래 빈칸을 **워크시트 '하네스 설계 캔버스'**에서 채운 뒤 사용하세요.

```
하네스 구성해줘. [내 업무: 무슨 일] 을(를) 돕는 팀이 필요해 — [필요한 역할들]. [원하는 산출물]을(를) 결과로 줘.
```

**예시(카테고리를 바꿔서):**
```
하네스 구성해줘. 카페 프랜차이즈 신메뉴 프로모션을 돕는 팀이 필요해 — 경쟁 프로모션을 조사하는 리서처와, 인스타 캡션을 쓰는 카피라이터. 경쟁 요약 반 페이지와 인스타 캡션 5개를 결과로 줘.
```

---

## 좋은 요청문 체크 (붙여넣기 전에)

- [ ] **도메인**: 무슨 일인지 한 줄로 들어갔나?
- [ ] **역할**: 누가 필요한지(2명 이상) 명시했나?
- [ ] **산출물**: 무엇을 몇 개/몇 페이지 받을지 적었나?

세 개가 다 있으면 하네스가 헤매지 않습니다.

---

## 문제가 생기면

| 증상 | 조치 |
|------|------|
| 팀이 아니라 단일 응답 | 에이전트 팀 기능 활성화 확인 |
| 트리거 안 됨 | "하네스 구성해줘" 정확히 입력, 플러그인 설치 확인 |
| 결과가 너무 길다 | 산출물 분량("1페이지", "5개")을 명시 |
| 품질이 아쉽다 | 톤/타겟을 덧붙여 재요청 (프롬프트 2번 참고) |
